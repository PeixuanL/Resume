# Trucker Path 业务一面准备 2026-07-13

## 这套资料怎么用

本文件夹是 HR 一面后的下一轮中文业务面准备。重点不是再背 HR 常规题，而是准备业务面可能会追的产品理解、项目经历、用户路径、设计协作、信息质量、UGC、指标和迁移能力问题。

## 文件结构

- `01_业务一面准备总纲_2026-07-13.md`：面试策略、核心答案、风险修正和反问问题。
- `02_产品矩阵与AI机会分析_2026-07-14.md`：结合 HR 提到的 7 个 project、B2C / B2B / Enterprise 分层、业务痛点和 AI 机会做的产品分析。
- `03_后续追问与项目介绍优先级_2026-07-14.md`：综合业务页面、HR 反馈、产品分析和简历项目经历，整理后续高概率追问、主项目 30/60/90 秒讲法和准备优先级。
- `TruckerPath_业务一面静态复习版_2026-07-13.html`：不用启动服务也能打开的静态复习页；已合入本轮中文业务面 mock 逐题优化版。
- `interactive-business-prep-v1/`：可 localhost 启动的互动练习器，支持随机抽题、计时、朗读、编辑保存和集中复习。

## localhost 启动

```bash
cd docs/面试/TruckerPath_业务一面准备_2026-07-13/interactive-business-prep-v1
node server.js
```

打开：

```text
http://127.0.0.1:8787/
```

如果 8787 被占用，可以用：

```bash
PORT=18788 node server.js
```

然后打开：

```text
http://127.0.0.1:18788/
```

## 本轮背诵优先级

1. 60 秒业务一面自我介绍。
2. Inspector 主项目：把分散信息整理成可执行、可反馈的产品闭环。
3. “我个人到底做了什么”：拆用户路径、产品对象、状态流、PRD/原型、MVP 和验收。
4. MVP 取舍：先跑通最小闭环，再做高级 AI/配置/体验增强。
5. Trucker Path 产品矩阵：B2C / B2B / Enterprise 背后的共同痛点是信息不确定。
6. 停车可信度：新鲜度、可信度、低摩擦贡献、信任信号。
7. UGC 机制：operational UGC，不是泛内容社区。
8. AI 机会：内部提效、司机端助手、信息质量治理、车队/Enterprise 助手，但要有解释性和人工确认。
9. 指标：不要只说 DAU/MAU，要绑定用户任务和信息质量。
10. 设计协作：从用户路径和关键决策点开始，不是直接从页面开始。
