## 结构化素材（背景—目标—约束—行动—结果—复盘）

### W1 · DataMesh 工作经历（总述）

| 要素 | 内容 |
|------|------|
| **背景** | 数字孪生公司，三条产品线（运维 Agent / 工业仿真 / MR 培训）并行 |
| **目标** | 资源向 0→1 运维 Agent 倾斜；大客交付与标品化并行 |
| **约束** | 3 条产品线、11 个版本并行；研发与客户成功资源有限 |
| **行动** | Roadmap 与优先级取舍；标品转化；POC 成功标准；发布 SOP |
| **结果** | 约 70% 定制转标品；发布周期 4 周→1–2 周；10+ 项目面客与交付 |
| **复盘** | 2025.10 战略聚焦后 Inspector 从项目制收敛为可复制底座（见 §9 时间口径） |

#### W1-1 产品规划与生命周期

| 要素 | 内容 |
|------|------|
| **背景** | 多产品线、多版本同时推进 |
| **目标** | 明确 Roadmap，向 0→1 运维 Agent 与大客交付倾斜 |
| **约束** | 11 个版本迭代窗口重叠 |
| **行动** | 负责版本 Roadmap 与优先级取舍 |
| **结果** | 资源向运维 Agent 底座及大客交付倾斜（E1） |
| **复盘** | 工作段不写 Agent 细节，细节归 P1 |

#### W1-2 业务梳理与标品转化

| 要素 | 内容 |
|------|------|
| **背景** | 汽车零部件灯塔工厂、头部通信运营商等大客定制需求多 |
| **目标** | 提取高频逻辑，降低重复研发 |
| **约束** | 客户差异大，需抽象共性 |
| **行动** | 梳理需求、优化功能架构、推动标品化 |
| **结果** | 约 70% 定制需求向标品转化（E1） |
| **复盘** | 面试 Q6 口述 Faurecia 等案例边界 |

#### W1-3 客户验证与项目落地

| 要素 | 内容 |
|------|------|
| **背景** | B 端长周期 POC 与售前并行 |
| **目标** | 面客验证、推动 POC 进入合同或规模化交付 |
| **约束** | 资源紧时需评估优先级 |
| **行动** | 面客调研、方案演示、定义 POC 成功标准；支撑售前 Demo 对齐 |
| **结果** | 10+ 项目参与；部分进入正式合同（E1） |
| **复盘** | 动词「支撑」售前，避免过度 claim 决策权 |

#### W1-4 流程优化与团队赋能

| 要素 | 内容 |
|------|------|
| **背景** | 标准资源包发布周期长 |
| **目标** | 缩短发布周期，赋能销售与交付 |
| **约束** | 跨产品线流程不统一 |
| **行动** | 制定 4 份发布指导与验收 SOP；重构线上发布流程 |
| **结果** | 4 周→1–2 周，约 75% 提效（E1） |
| **复盘** | 与 P1 实施周期压缩同体系，简历不重复写 Agent |

### P1 · AI 设施运维产品（0→1）

| 要素 | 内容 |
|------|------|
| **背景** | 设施运维规模化复制难；告警与处置链路分散 |
| **目标** | 0→1 孵化标准运维底座与 Agent 闭环 |
| **约束** | 多产品线并行；矩阵数据串联需跨团队 |
| **行动** | Agent 分工 + RAG；标品架构收敛；矩阵 Demo 拍板优先级 |
| **结果** | MVP 覆盖高优诉求 80%+；5 家+ 潜在商机；运营商周期 5–6 月→1–2 月；新加坡数百万级方案（E1） |
| **复盘** | 项目时间 2025.10 起与入职 2024.12 不矛盾（见进度 doc §9） |

---

### P2 · AI 辅助工业仿真编辑产品

| 要素 | 内容 |
|------|------|
| **背景** | 产线仿真搭建门槛高，缺可复盘数据闭环 |
| **目标** | AI 辅助决策 + 数据闭环 + 搭建提效 |
| **约束** | 非全程模型侧；Prompt 与业务对齐为主 |
| **行动** | Prompt 分层与输出约束；3D 可视化映射；对接工业仿真软件 |
| **结果** | 数据闭环进入正式交付；跨境 POC→合同（E1/E2 定性） |
| **复盘** | 不写 Agent/RAG 主导；强度低于 P1 |
