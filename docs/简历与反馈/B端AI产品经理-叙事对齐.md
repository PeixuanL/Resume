## 全篇主线（选定）

**主主线（单选）：** **0→1 → 规模化 AI 能力落地**

- 摘要与最近项目突出：设施运维 **Agent 底座** 从场景验证 → 标品化 → POC/商用 → 多产品线复制。
- 工作段承接：资源向 0→1 运维 Agent 倾斜、标品转化、POC 成功标准、交付 SOP。

**辅线（归类用，不单列第二主线）：** **复杂 B 端场景下的产品治理**

- 多产品线（3 条 / 11 版本）、大客定制与优先级取舍、跨职能（研发 / 客成 / 美术 / 售前）接口。
- 不写第二条平行主线，避免招聘方「不知道你到底是平台人还是 0-1 人」。

**未选主线及原因**

| 候选主线 | 不作为主线的理由 |
|----------|------------------|
| 平台/中台型 AI 产品 | 标品化、70% 转化是**手段**，已并入主主线「规模化落地」 |
| 纯复杂场景需求治理 | 缺 AI 差异化，不适合 B 端 **AI** PM 首屏 |

## 模块顺序（相对旧稿调整）

公约框架稿采用下列顺序（**核心能力下移至工作经历、项目之后**）：

1. 页眉（求职意向 + 联系方式）
2. 专业摘要（原「个人总结」）
3. 工作经历（主体，50%+ 视觉权重）
4. 项目经历（代表项目，与工作经历去重）
5. 核心能力（维度化，指向经历）
6. 教育背景
7. 技能清单

**对应 HTML：** `模板/B端AI产品经理_简历_框架.html`；内容实例化见 `草稿/当前/李沛宣_B端AI产品经理_简历_单页_公约框架稿.html`。

**八维竞争力 × 主线映射（摘要 / 核心能力 / 经历须闭环）：**

| 维度 | 主线上如何体现 |
|------|----------------|
| 需求取舍与优先级 | 工作 Roadmap；P1 标品收敛 |
| 数据与指标闭环 | 70%、75%、80%、周期压缩 |
| AI 边界与落地 | 摘要 + P1 Agent/RAG/兜底 + P2 Prompt |
| 跨职能协作 | 工作面客/售前；P1 矩阵 Demo |
| 风险合规（产品侧） | 人机协同、可解释、人工确认 |
| 系统化/平台化 | 标品化、通用模块与实施基线 |
| 贡献边界一致 | 负责/定义/推动/拍板/支撑 梯度 |
| 商业化结果 | POC→合同、商机、数百万级方案 |

详见 [`B端AI产品经理-经历素材对齐.md`](B端AI产品经理-经历素材对齐.md) §八维竞争力。

## 术语与职级强度（全篇统一）

| 统一用语 | 避免 |

## 代表项目簇（2–4 个，去重边界）

| 簇 | 对外标题 | 与工作段关系 |
|----|----------|--------------|
| **P1** | AI 设施运维产品（0→1） | 工作段只写精力分配与标品/POC/流程；**Agent/RAG/闭环细节仅在 P1** |
| **P2** | AI 辅助工业仿真编辑产品 | 工作段不展开 Prompt/仿真；**P2 独占 Designer 向 AI 表述** |

全篇 **2 个** 深度项目簇即可；不再增加第三项目块（除非新增 E0/E1 经历）。
