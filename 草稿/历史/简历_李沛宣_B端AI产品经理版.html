重点项目

Inspector｜数字孪生设施运维底座平台

产品经理｜2025.10 — 至今

背景：商业楼宇 / 工业运维项目中，设备告警、BMS 曲线、历史工单、SOP、图纸与现场经验分散，传统系统只能展示“发生了什么”，难以解释根因、影响范围和下一步处置。

  • 目标：将 Inspector 从项目型工具升级为可复用的设施运维底座，支撑海外标杆客户交付与后续规模化复制；产品目标不仅是展示告警，而是形成“告警—诊断—方案—工单—执行—报告—知识沉淀”的闭环。
  • 平台能力拆解：盘点过往定制项目，规划 3D 可视化沙盘、资产追踪、告警可视化、复杂工单流转、维保 SOP 模板等模块，并把设备、空间、系统、告警、工单和实施流程定义为可复用的产品基线。
  • AI Agent / RAG 方案：设计“告警理解 Agent + 上下文检索 / RAG + 诊断推理 + 方案生成 + 工单草稿 + 知识沉淀”的链路，明确 Agent 检索设备台账、空间位置、BrickSchema 关系、BMS / IoT / DFS 数据、历史工单和 SOP 后，再输出可能根因、影响范围、建议动作和工单字段。
  • 可靠性与风险控制：针对电梯、消防、强电等高风险场景，规划置信度、证据引用、缺失信息提示、人工审核、专家升级和反馈回流机制；AI 只生成初步诊断 / 方案 / 工单草稿，正式下发和事故报告归档必须经过人工确认。
  • 可运行验证与 PRD 回写：在既有前后端代码仓与 UI 规范下，使用 Codex 围绕工单闭环、设备维保 SOP 模板库和界面规则做可运行验证，将接口变化、交互约束、字段规则和验收标准回写到 PRD，降低研发理解偏差。
  • 结果：MVP 覆盖某海外标杆大客 80%+ 高优先级标准诉求，吸引 5 家以上客户跟进；基于该底座支撑某头部通信运营商三期项目，将实施周期由 5–6 个月压缩至 1–2 个月。

Designer｜虚拟规划与仿真编辑平台

产品经理｜2024.12 — 至今

背景：工业 / 建筑客户已有大量场景和仿真数据，但孪生场景搭建门槛高、物理仿真能力不足，导致客户难以用 3D 结果回看产线或空间运行问题。

  • 目标：提升复杂孪生场景的搭建效率和仿真可信度，让客户能够复用既有数字资产，并把仿真记录转化为可分析、可展示、可复盘的 3D 结果。
  • 仿真能力与外部对接：设计自主寻路、实时碰撞监测、物理重力模拟等能力，推进与外部仿真软件的数据对接方案，减少重复建模与场景制作成本。
  • 数据闭环与指标呈现：打通 Designer 与 DFS 数据平台的数据流转，串联“记录生成—数据分析—3D 结果回顾”链路；设计运行路径、热力图等 3D 可视化映射方案，辅助客户识别产线瓶颈和布局优化机会。
  • 交互与 AI 预研：优化传送带、机械臂等编辑交互，提升数据驱动灵活性；参与“AI 理解孪生场景”预研，补充孪生体属性、模板与行为树等语义信息，使后续 Agent 能够理解空间结构、设备关系和行为逻辑。
  • 结果：支撑日本头部建工企业项目由 POC 转为正式合同落地，并沉淀可复用的场景搭建、数据接入与仿真展示方案。
