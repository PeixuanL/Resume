专业摘要

在工业互联网与设施运维等复杂 B 端场景下，专注大模型应用产品化：负责场景定义、Agent 工作流与人机协同验收，把 AI 做成可解释、可兜底、可迭代交付的能力。 0→1 主导设施运维 Agent 底座，推动 POC 进入商用与多产品线复制交付。

工作经历

DataMesh B 端产品经理 2024.12 - 至今

战略重心：设施运维 Agent 底座（0→1）；并行工业仿真编辑迭代与大客交付；MR 培训线维护与售前资产。

  • 产品规划与生命周期：在三条产品线、多版本并行约束下，负责 11 个版本迭代的 Roadmap 与优先级取舍，推动资源向 0→1 运维 Agent 底座 及大客交付倾斜。
  • 业务梳理与标品转化：围绕汽车零部件灯塔工厂、头部通信运营商等行业大客，提取高频可复用逻辑并优化功能架构，实现约 70% 定制需求向标品转化，降低重复研发与沟通成本。
  • 客户验证与项目落地：负责 10+ 个项目的面客调研、方案演示与交付；定义 POC 成功标准，推动部分项目进入正式合同或规模化交付；资源紧时评估优先级并提供替代方案；支撑头部汽车与物流大客售前方案与 Demo 对齐。
  • 流程优化与团队赋能：制定 4 份发布指导与产品验收 SOP，重构标准资源包线上发布流程，将周期由 4 周缩短至 1–2 周（交付提效约 75%）；沉淀跨产品线交付清单，赋能销售与交付复用。

核心能力

  • 场景定义与 Agent 工作流：拆解 B 端业务场景与数据依赖，设计 Agent 分工、RAG/知识库 联动与 Prompt/输出约束，建立可验收主链路。
  • 评测验收与可控交付：定义人机协同验收标准与兜底策略，约束可解释输出与人工确认节点，按场景迭代评测与上线节奏。
  • 平台化与产品路线：将大客户共性沉淀为通用模块与实施基线，而非逐项目做加法；统筹 Roadmap 与功能优先级。
  • POC 验证与规模化落地：收敛 POC/MVP 范围与成功标准，推动验证结果转化为复制交付与商用落地。
