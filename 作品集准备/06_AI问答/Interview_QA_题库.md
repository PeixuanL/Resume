# Interview Q&A 题库

更新时间：2026-06-29

## 定位

这是后续作品集网站的可选加分模块。第一版先作为面试准备材料，不急着做成交互聊天。

## 问题清单

| 问题 | 回答状态 | 备注 |
|---|---|---|
| 你在主案例中最大的产品挑战是什么？ | 待写 | 应围绕业务闭环和多角色协同 |
| 为什么不让 AI 自动派单或自动维修？ | 待写 | 强调风险、人机确认、审计 |
| 你如何定义这个项目的 MVP？ | 待写 | 强调最小真实闭环 |
| 你如何验证这个方案有效？ | 待写 | 用 POC/评审/反馈替代虚构数据 |
| 你在项目中具体负责什么？ | 待写 | 需要用户补个人贡献边界 |
| 你如何和研发/算法/交付协作？ | 待写 | 强调需求澄清、状态流、验收标准 |
| 如果重做一次，你会怎么优化？ | 待写 | 可讲数据闭环、知识库质量、状态追踪 |
| Vibe Coding 在作品集里怎么体现？ | 待写 | 写成产品验证和原型迭代方法 |

## 回答边界

1. 只基于脱敏材料回答。
2. 不编造 ROI、准确率、成本节约。
3. 不把方案/原型写成完整上线。
4. 不把 AI 写成无人审核的自动执行者。
