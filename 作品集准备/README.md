# 作品集准备工作区

本文夹用于集中管理李沛宣 B 端 AI 产品经理作品集相关内容，后续作品集推进状态、案例材料、PDF/PPT、网站草稿、AI 问答内容都放在这里。

## 当前定位

**目标作品集：** 面向 B 端 AI 产品经理 / 工业数字孪生 / 智能楼宇 / 设施运维方向的求职作品集。

**核心叙事：** 不是泛泛展示多个项目，而是用一个主案例证明复杂 B 端场景的 0-1 产品判断、AI 能力落地、人机协同闭环和产品验证方法。

## 内容边界

| 层级 | 内容 | 作用 |
|---|---|---|
| 主案例 | Inspector / 设施运维 / 告警工单闭环 / AI 辅助诊断 | 证明 0-1、B 端复杂场景、AI 产品化能力 |
| 支撑案例 | Designer / DFS / Singapore FDS / 英文产品文档支持择一 | 证明跨系统、数据/空间底座、国际化或交付支持能力 |
| AI-Native 方法 | Cursor / Codex / Kimi 辅助 PRD、原型、案例梳理 | 证明用 AI 工具提升产品工作效率，但不包装成生产交付成果 |

## 推进原则

1. 先内容可投递，再形式 AI-Native。
2. 先主案例，再支撑案例。
3. 先证据边界，再写漂亮表达。
4. 不虚构 ROI、准确率、节省成本、完整上线状态。
5. AI 写成能力层：知识检索、诊断建议、流程辅助、人机确认，不写成自动替代人。
6. Vibe Coding / Codex / Cursor 写成产品验证和原型迭代方法，不写成公司生产交付成果。

## 每次更新方式

每次推进后至少更新：

1. `00_推进管理/推进状态.md`
2. 当前正在编辑的案例或材料文件
3. 如有新增截图/产物，补到 `01_材料清单/材料证据清单.md`
