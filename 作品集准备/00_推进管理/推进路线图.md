# 作品集推进路线图

## 目标

2 周内完成可投递版本：**一份 PDF/PPT 作品集 + 一个轻量网页入口草案**。

4 周内完成增强版本：**可访问网站 + 主案例深页 + 支撑案例 + 可选 AI 问答模块**。

## Phase 0：建立工作区与状态机制

时间：2026-06-29

**产出：**

- 统一文件夹：`作品集准备`
- 推进状态看板
- 材料证据清单
- 主案例工作稿

**完成标准：**

- 后续作品集材料不再散落在多个位置。
- 每次推进都能看到当前状态和下一步。

## Phase 1：主案例内容定稿

建议时间：2026-06-29 至 2026-07-03

**目标：** 把 Inspector / 设施运维案例讲成一个业务闭环故事。

**主案例结构：**

1. 业务背景：设施运维为什么复杂。
2. 用户角色：运维人员、设施主管、工程师、客户管理员。
3. 核心问题：不是缺一个 AI，而是缺从异常到处置的闭环。
4. 产品方案：告警/设备状态 -> AI 辅助解释/知识检索 -> 处置建议 -> 人工确认 -> 工单流转 -> 记录沉淀。
5. 关键判断：AI 辅助而非自动替代；高风险动作保留人工确认；MVP 先证明最小真实闭环。
6. 验证依据：POC、评审反馈、用户反馈、方案沉淀、可复用模块。

**完成标准：**

- 能讲 8-10 分钟。
- 能回答“你具体负责什么”“为什么这么设计”“AI 放在哪里”“怎么验证有效”。

## Phase 2：第一版 PDF/PPT

建议时间：2026-07-04 至 2026-07-07

**建议页数：** 10-12 页。

| 页码 | 内容 |
|---|---|
| 1 | 个人定位 + 作品集目录 |
| 2 | 主案例一页总览 |
| 3 | 业务痛点与用户角色 |
| 4 | 原始流程问题 / Before |
| 5 | 产品闭环设计 / After |
| 6 | AI 辅助能力放在哪里 |
| 7 | 关键页面 / 原型 / 信息架构 |
| 8 | 产品判断与取舍 |
| 9 | 验证方式与阶段性成果 |
| 10 | 支撑案例概览 |
| 11 | AI-Native 工作方式 |
| 12 | 总结：能力标签与适配岗位 |

**完成标准：**

- 可以作为附件发给面试官。
- 不依赖网站也能完整表达主案例。

## Phase 3：轻量作品集网站

建议时间：2026-07-08 至 2026-07-14

**信息架构：**

```text
Home
- 一句话定位
- 主案例入口
- 支撑案例入口
- Resume / Contact

Case: Facilities Ops AI Workflow
- Business Context
- User & Scenario
- Product Loop
- AI Capability Layer
- Screens / Prototype
- Validation & Reflection

Case: Support Project
- Scope
- Contribution
- What it proves
```

**完成标准：**

- 手机和电脑都能打开。
- 3 分钟内能看懂你是谁、主案例是什么、强项在哪里。

## Phase 4：AI Interview Q&A

建议时间：2026-07-15 之后

**定位：** 加分项，不影响第一版投递。

**形式建议：** 点击问题展开回答，或基于固定材料的轻量问答，不做不可控开放聊天。

**完成标准：**

- 回答全部来自脱敏案例材料。
- 不虚构数据，不泄露敏感信息。
