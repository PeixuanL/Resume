## 一、公司背景与产品上下文

### 1.1 DataMesh是谁

DataMesh（商询科技）成立于2014年，定位为**工业数字孪生基础设施提供商**，帮助企业构建数字化运营能力。核心产品矩阵：

| 产品 | 定位 |
|---|---|
| **FactVerse** | 工业数字孪生快交付平台，主打低代码/无代码3D场景编排 |
| **Data Fusion Services (DFS)** | 企业级数据集成平台，打通IoT/ERP/MES等数据源 |
| **Inspector** | 企业级智能运维监控平台，数字孪生+空间计算+AI |
| **Director** | XR培训指导协同平台 |
| **Checklist** | 流程操作助手（MR+AI） |
| **Simulator** | 仿真模拟器 |

**典型客户：** 亿纬锂能（数字孪生工厂运营平台）、大林组（建筑施工流程优化）
**业务覆盖：** 中国大陆、日本、东南亚、北美

### 1.2 DataMesh的运营特征

理解DataMesh的运营特征，是写好这份PRD的前提：

| 维度 | 特征描述 |
|---|---|
| **业务本质** | 产品公司 + 项目交付双轨并行 |
| **项目类型** | 数字孪生交付项目（3D场景编排+IoT数据接入+系统集成），周期3-18个月 |
| **团队结构** | 销售+解决方案（SE）+产品研发+实施交付+运维，四条线并行 |
| **核心矛盾** | 产品研发节奏 vs 项目定制交付，人力是核心资源 |
| **财务口径** | 项目成本需精确到人天，项目收入按里程碑确认，需匹配财务核算周期 |
| **数据特征** | IoT数据、时间序列数据、3D模型数据、实时状态数据 |

## 二、产品定位

### 2.1 核心痛点

DataMesh内部运营目前面临的真实痛点：

**业务端：**
- 商机→报价→立项之间数据断层，签约后才知道项目预估成本
- 实施团队不清楚项目还有多少预算可以使用，边做边超
- 不同项目的实施方法论不统一，知识经验散落在个人手里

**财务端：**
- 项目工时由人工填报纸质/Excel，财务月底才能算出项目成本，滞后至少一个月
- 项目研发费用（产品研发）和项目实施成本混在一起，无法精确拆分
- 销售提成、项目奖金没有系统化依据，依赖人工核算

**管理层：**
- 老板想看"这个项目今天赚不赚钱"，系统里查不到
- 销售预测和项目交付进度脱节，无法预测未来收入

**核心诉求一句话：**

> 让业务部门和财务部门在同一套系统里工作，项目从签合同第一天开始，到交付完成，每一天的成本和利润都清清楚楚。

### 2.2 产品愿景

> **DataMesh 内部业财一体化管理平台 —— 让项目交付与财务核算实时联动，让每个决策者随时看清项目经营真相。**

三个核心价值：
- **业务财务一张网** — 销售/实施/产品/财务在同一平台，数据互通
- **项目经营实时化** — 从签约到结算的全生命周期，利润实时可见
- **管理决策数据化** — 收入预测/成本管控/资源调度，基于数据而非直觉

### 4.1 销售管理域

#### 4.1.1 功能概述

销售管理域是业财一体系统的起点，实现从线索获取到合同签约的全流程管理，并与财务系统无缝对接。

```
线索获取 → 商机录入 → 需求确认 → 方案制作 → 报价生成 → 报价审批 → 合同签约 → 项目立项
     │           │           │           │           │           │           │
     └───────────┴───────────┴───────────┴───────────┴───────────┴───────────┘
                                       销售漏斗可视化
```

##### 4.1.3.3 销售漏斗计算规则

```
漏斗金额 = Σ(商机预估金额 × 赢单概率)

各阶段赢单率：
- 线索阶段：10%
- 需求确认：30%
- 方案阶段：50%
- 报价阶段：80%
- 签约阶段：100%

漏斗转化率 = 本阶段商机数 / 上阶段商机数 × 100%
```

| 销售漏斗预测 | 预测各阶段转化率和最终签约金额 | P1 | |
| 预测 vs 实际 | 对比销售预测与实际签约，分析偏差原因 | P2 | |
| 销售目标追踪 | 追踪年度销售目标完成进度 | P1 | |

##### 4.1.5.2 预测计算模型

```
预测签约额 = Σ(各阶段商机预估金额 × 赢单概率)

月度预测签约额 = Σ(本月预计签约商机 × 概率) + Σ(下月预计签约商机 × 概率 × 转化系数)

转化系数参考历史数据：
线索→需求确认：30%
需求确认→方案：60%
方案→报价：70%
报价→签约：80%
```

##### 4.2.5.3 成本归集计算公式

```
人工成本 = Σ(工时小时数 × 人员日费率 / 8)

日费率来源优先级：
1. 项目团队成员配置的日费率
2. 岗位默认日费率（系统配置）
3. 月薪 / 22天（兜底计算）

成本归集规则：
- 内部员工工时：归集到人工成本
- 外部协作者工时：归集到外包成本
- 工时填报日期 = 成本归集日期（不以审批日期为准）

每日凌晨2点执行成本归集任务：
1. 获取昨日已审批工时
2. 按人员费率计算各项成本
3. 汇总写入项目成本明细表
4. 更新项目实际成本和利润
```

##### 4.3.4.3 成本归集规则

```
成本归集原则：以业务发生日期为准，不以付款日期为准

人工成本：
  归集日期 = 工时填报日期
  归集金额 = 工时小时数 × 人员日费率 / 8
  归集方式 = 自动归集（每日凌晨计算）

外包成本：
  归集日期 = 结算单签字确认日期
  归集金额 = 结算单金额
  归集方式 = 手动录入

差旅成本：
  归集日期 = 差旅报销审批通过日期
  归集金额 = 报销金额
  归集方式 = 自动关联（报销时选择项目）

硬件/云资源：
  归集日期 = 采购订单审批通过日期
  归集金额 = 采购金额
  归集方式 = 采购入库时自动关联

项目利润计算：
  项目利润 = 已确认收入 - 已归集成本
  项目毛利率 = (已确认收入 - 已归集成本) / 已确认收入 × 100%
```

##### 4.3.4.4 预算预警规则

| 预警级别 | 触发条件 | 预警方式 | 处理方式 |
|----------|----------|----------|----------|
| 绿色 | 执行率 ≤ 80% | 不预警 | 正常 |
| 黄色 | 80% < 执行率 ≤ 100% | 钉钉+站内通知项目经理 | 关注进度 |
| 红色 | 执行率 > 100% | 钉钉+邮件通知PMO+财务 | 必须调整预算或控制成本 |

│                           ▼                                          │
│                    ┌─────────────┐                                  │
│                    │  利润预算   │                                  │

│                    │  预算审批   │  财务→总经理→董事会             │
│                    └──────┬──────┘                                  │

## 七、财务核算规则（关键约定）

> ⚠️ 以下规则是业财一体系统设计的核心依据，产品经理和开发团队必须深入理解。

### 7.1 收入确认规则

```
数字孪生交付项目（里程碑付款）：
  收入确认时点 = 合同约定里程碑达成 + 客户签字确认
  确认金额 = 里程碑对应产值金额

举例：
  合同总额100万，分4期：
  - 合同签署：收到预付款30万（→ 合同负债）
  - 方案冻结：确认收入25万（→ 应收账款）
  - UAT通过：确认收入30万（→ 应收账款）
  - 验收通过：确认收入15万（→ 应收账款）
  质保金10万（质保期满后确认）
```

### 7.2 成本归集规则

```
人工成本：
  人工成本 = Σ(每个员工当日项目工时 × 该员工日费率)
  日费率 = 月薪 ÷ 22天（可配置）

外包成本：
  按实际结算单日期归集，不以付款时间为准

硬件/云资源：
  采购订单审批通过即归集成本，不以到货/开票为准
```

### 7.3 项目利润计算

```
项目毛利率 = (项目总收入 - 项目总成本) / 项目总收入 × 100%

项目总成本 = 人工成本 + 外包成本 + 差旅成本 + 硬件采购 + 云资源 + 其他直接成本

注意：
  管理费用（房租/行政/HR等）不计入项目成本，仅计入公司整体损益
  研发费用单独归集，不并入项目成本
```
