# ADR-004: 领域专家智能体库（Expert Agent Library）

## 状态：已接受

## 背景
系统需要具备领域专家智能体能力。不同的设施设备管理场景需要不同领域的专业知识：
- 暖通空调（HVAC）专家
- 电气系统专家
- 电梯/垂直运输专家
- 水泵/流体系统专家
- 能效优化专家

这些专家角色：
1. 由 AI 设计，具备该领域最强的能力描述
2. 用户可以自行创建/配置新的专家角色
3. AI 在工作时遵从相关专家的指示和判断标准

## 决策

### 核心概念：Expert Agent = 结构化领域知识 + AI Prompt

每个专家智能体包含：
- **身份定义** — 名称、领域、专业描述
- **能力声明** — 该专家擅长什么（system prompt 核心）
- **诊断规则** — 领域特定的判断标准和阈值
- **决策框架** — 面对特定情况时的处理流程
- **知识参考** — 关联的文档、手册、标准

### 工作机制
```
用户创建/选择专家 → 专家 Profile 注入 AI Context
                      ↓
传感器异常 → AI 引擎 + 专家 Prompt → 专业诊断结论
                      ↓
诊断结论 → 生成工单（包含专家建议的维修步骤）
```

### 数据模型
- `expert_agents` — 专家定义表
- `expert_knowledge_rules` — 专家的诊断规则
- `expert_equipment_bindings` — 专家与设备的绑定关系

## 后果
- 系统具备可扩展的领域智能
- 用户无需 AI 专业知识，只需描述领域需求
- AI 可以"自举"生成新专家
