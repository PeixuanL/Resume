## 二、现状痛点量化分析

### 2.1 能耗问题

| 痛点 | 现状数据 | 目标值 | 差距 |
|------|---------|--------|------|
| 供热单耗 | 45~65 W/㎡（部分社区远超标） | ≤50 W/㎡ | 降低9.1%+ |
| 二次管网电耗 | 1.8~2.0 kWh/㎡ | ≤1.0 kWh/㎡ | 降低44%+ |
| 管网热损失率 | >12%（部分路段） | ≤8% | 降低33%+ |
| 换热器效率 | <80%（梵谷E1站） | ≥90% | 提升12%+ |
| **年额外成本** | **天然气浪费679.77万 + 电费超支72.78万 = 752.55万元/年** | | |

**量化损失**：按能耗年均自然增长2%测算，2026-2030年若不改造，累计额外成本将超**1000万元**。

### 2.2 调控问题

| 痛点 | 现状数据 | 目标值 | 标准要求 |
|------|---------|--------|---------|
| 调度模式 | 人工经验调度 | AI自主调度 | — |
| 水力失调率 | **30%** | ≤15%（2026）→≤10%（2028） | — |

| 室温满意度 | **85%** | ≥98% | 朝阳区考核：≥90% |
| 室温投诉占比 | **42%** | — | — |
| 预测准确率 | 75-80%（传统经验） | ≥95%（AI多维预测） | 北京市要求：≥90% |

**量化损失**：
- 社会责任考核扣分 → 间接损失约**80万元/年**
- 故障响应不及时 → 赔偿+抢修额外支出约**30万元/年**

### 2.3 数据孤岛问题

**8套系统互不连通**：

| 系统 | 数据类型 | 问题 |
|------|---------|------|
| SIS系统 | 配电站/热表数据 | 与调度系统不通 |
| PVSS系统 | 换热站运行数据 | 无AI联动 |
| 投诉客服系统 | 报修/用户反馈 | 人工处理 |
| 收费系统 | 热费/用户信息 | 账单争议多 |
| 其他4套系统 | 各类运营数据 | 数据割裂 |

**后果**：
- 无法向市级城市运行管理平台实时上传数据，已收到**2次整改通知**
- 人工整理上报数据，误差率**15%**，年人力成本**20万元**
- 缺乏能碳管理能力，无法按国标进行能碳核算

### 2.4 服务体验问题

- 热费结算争议多发
- 老年用户占比**30%**，智能化服务适配性差
- 传统人工服务模式无法满足多元化需求
- 商业用户要求恒温误差≤±1℃，2025年供热不稳定导致投诉占商业用户总量**70%**

**FR-ML-002：数据清洗流水线**

```
原始数据 → 格式校验(Schema Validation)
         → 缺失值处理(KNN填充/均值/中位数/前向填充, 按字段配置)
         → 异常值检测(3σ法/IQR法/业务规则, 如温度>150℃或<-40℃判定无效)
         → 重复数据去除(时间戳+设备ID联合去重)
         → 数值标准化(Min-Max归一化/Z-Score标准化, 按模型需求选择)
         → 类别编码(One-Hot/Label Encoding)
         → 时间特征提取(小时/星期/月份/节假日/供暖季标识)
         → 滑动窗口(24h/72h/168h窗口特征聚合)
         → 质量评分(完整性/一致性/时效性打分, 总分≥98%方可入库)
         → 训练集版本化存储(带元数据标签, 可回溯任意版本)
```

**数据质量监控规则（具体阈值）**：

| 监控维度 | 规则 | 告警阈值 | 处理方式 |
|---------|------|---------|---------|
| 完整性 | 字段非空率检查 | <95% | 黄色预警，触发数据补采 |
| 一致性 | 跨系统数据比对 | 差异>2% | 红色预警，人工核查 |
| 时效性 | 数据延迟检查 | >30秒 | 黄色预警，检查网关状态 |
| 准确性 | 物理约束校验(如回水温度<供水温度) | 违反即报 | 标记无效+回溯传感器 |
| 唯一性 | 重复记录检测 | >0.1% | 自动去重+告警 |

#### 4.1.3 特征工程模块

**FR-ML-003：供热领域专用特征算子库（≥30个）**

| 算子分类 | 算子名称 | 计算逻辑 | 输入 | 输出 |
|---------|---------|---------|------|------|

| | 风寒指数算子 | WCI = 13.12+0.6215T-11.37V^0.16+0.3965TV^0.16 | 温度T/风速V | 风寒指数 |
| | 湿度修正算子 | Q_修正 = Q_base × (1 + 0.02×(RH-50)) | 基础负荷/相对湿度 | 修正负荷 |
| | 72h天气趋势算子 | 线性拟合未来72h温度变化斜率 | 预报温度序列 | 趋势斜率 |
| 建筑热工 | 围护结构热阻算子 | R = Σ(δᵢ/λᵢ) | 各层材料厚度/导热系数 | 总热阻 |
| | 建筑热惰性算子 | D = Σ(Rᵢ×Sᵢ) | 热阻/蓄热系数 | 热惰性指标 |

| 用户行为 | 入住率算子 | 用热户数/总户数 | 收费系统数据 | 入住率(0-1) |
| | 用热习惯聚类 | K-Means聚类用户日用热曲线 | 历史用热数据 | 用户类型标签 |

| 时序特征 | 周期分量提取 | FFT提取日周期/周周期分量 | 负荷时间序列 | 周期特征向量 |
| | 滞后特征 | lag_1h, lag_24h, lag_168h | 负荷序列 | 滞后值 |
| | 移动平均/标准差 | MA(24h), STD(24h) | 负荷序列 | 统计特征 |

**低代码建模界面需求**：
- 拖拽式操作：从算子库拖拽特征算子到画布 → 连接数据源 → 选择算法 → 设置超参数（滑块/输入框）→ 一键训练
- 算法选型下拉：线性回归/XGBoost/随机森林/LSTM/GRU/CNN/PPO/IsolationForest等≥15种
- 自动调参：网格搜索/贝叶斯优化/随机搜索可选，运行时展示实时收敛曲线
- 训练进度：进度条+预估剩余时间+GPU/CPU使用率实时展示

#### 4.1.4 模型评估与版本管理

**FR-ML-004：模型评估**

| 评估类型 | 指标 | 可视化输出 |
|---------|------|-----------|
| 回归模型 | MAE/RMSE/MAPE/R²/最大绝对误差 | 残差分布图、预测值vs实际值散点图、误差随时间变化折线图 |
| 分类模型 | Accuracy/Precision/Recall/F1/AUC-ROC | 混淆矩阵热力图、ROC曲线、PR曲线 |
| 通用 | 学习曲线(训练集vs验证集) | 过拟合/欠拟合诊断图 |
| 可解释性 | SHAP全局特征重要性 | 蜂群图、瀑布图（单预测解释） |
| 可解释性 | LIME局部解释 | 单条预测的特征贡献柱状图 |

**模型版本管理（MLflow集成）**：
- 每次训练自动记录：超参数、训练数据版本hash、评估指标、训练时长、GPU/CPU消耗
- 模型状态：`Staging`(开发中) → `Production`(生产) → `Archived`(归档)
- A/B测试：新模型上线时自动灌入10%流量对比，72h后指标优于旧版方可全量切换
- 金丝雀发布：逐步放量(10%→30%→50%→100%)，每阶段≥24h观察
- 紧急回滚：一键回滚至上一`Production`版本，回滚时间≤2分钟

| 自动扩缩容 | 支持，2min内完成 |


| 电力 | 0.581 kgCO₂/kWh | 北京市电网排放因子(DB11/T1166-2015) |

**核算公式**：
```
范围1(直接排放) = V_gas × EF_gas   (天然气消耗量×排放因子)
范围2(间接排放) = E_elec × EF_elec  (电力消耗×排放因子)
总排放 = 范围1 + 范围2
碳减排量 = 基准线排放 - 实际排放  (基准线=改造前3年平均值, 含气象修正因子)
```

**碳资产管理时间线**：
- 2026年：建立碳核算体系+月度报告
- 2027年：碳资产确权+减排量备案登记
- 2028年：北京环境交易所开户
- 2029年：分批次碳减排量挂牌交易（预计年交易额~48万元，按60元/吨）

## 十三、建设原则

1. **标准规范**：遵循GB/T51350-2019、CJJ28-2014等国标 + 北京电控内部规范
2. **先进适用**：技术版本5年不落后（Spark3.0/Flink1.18/TF2.15），优先国产设备降本10-15%
3. **开放标准**：核心技术栈开源（Hadoop/Spark/LangGraph），避免厂商锁定；接口标准化（RESTful/OPCUA）
4. **国产化适配**：麒麟/统信OS + 鲲鹏/飞腾芯片 + 信创超融合平台
5. **模块化设计**：数据采集/模型训练/智能决策/设备控制独立模块，支持单独升级
6. **弹性扩展**：硬件集群化扩容 + 软件插件化架构，预留50%扩展空间
7. **安全可靠**：AES-256 + RBAC + 双机热备 + 等保三级，MTBF≥99.9%
