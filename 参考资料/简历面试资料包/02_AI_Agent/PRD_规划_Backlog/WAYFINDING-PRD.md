# 室内寻路导航系统 — 产品需求文档 (PRD)

> 版本: 1.0 | 日期: 2026-02-14 | 状态: 草案

---

## 1. 产品愿景

**"让每位访客都能轻松找到目的地。"**

在大型商业综合体、交通枢纽、旅游景区等复杂室内环境中，访客常常因为楼层多、通道复杂、标识不清而迷路，浪费大量时间并产生负面体验。本产品旨在为 FactVerse 数字孪生平台提供一套完整的室内寻路导航能力，让用户通过手机扫码或触屏终端即可获得从当前位置到目的地的最优路径指引，包括跨楼层导航、无障碍路线、多语言支持等核心能力。

### 产品定位

- **基盘能力**：导航图引擎、多层路由、路径渲染、定位接口 — 通用且可复用
- **模組能力（WayfindingOps）**：针对旅游场景、群组导航、热力分析、紧急疏散等垂直需求的增值模块

### 目标用户群

| 用户类型 | 场景 |
|---------|------|
| 旅游场景客户 | 景区、展馆、文化设施的访客导览 |
| 商业综合体客户 | 购物中心、会展中心的店铺寻路 |
| 交通枢纽客户 | 机场、车站的登机口 / 站台引导 |
| 办公园区客户 | 写字楼、工业园区的访客指引 |

## 2. 用户故事

### 2.1 游客 — 小王（自由行旅客）

> 作为一名外国游客，我希望在到达一个大型展馆后，用手机扫描入口的二维码，即可获得英文界面的室内地图，搜索我想参观的展厅，系统为我规划最优路线并提供逐步导航指引，这样我就不会在陌生环境中迷路。

**验收标准：**
- 扫码后 3 秒内加载导航界面
- 支持中文、英文、日文搜索 POI
- 路径规划响应时间 < 500ms
- 到达每个转弯点时自动更新指引

### 2.2 运营人员 — 李经理（场馆管理员）

> 作为场馆运营人员，我希望能通过后台编辑器维护 POI 信息（名称、分类、营业时间、多语言描述），管理导航节点和路径权重，并查看访客热力图来优化动线设计。

**验收标准：**
- 可视化编辑导航图（拖拽节点、连线）
- 批量导入/导出 POI 数据
- 实时热力图展示（延迟 < 30 秒）

### 2.3 紧急疏散 — 消防安全员

> 作为安全管理员，在紧急情况下我需要系统自动切换为疏散模式，为所有在场访客推送最近出口路线，并根据被封锁通道动态调整路径。

**验收标准：**
- 一键触发疏散模式
- 可标记封锁区域，路径自动绕行
- 疏散路径计算 < 200ms

### 2.4 无障碍用户 — 轮椅使用者张先生

> 作为轮椅使用者，我需要系统为我规划无障碍路线，避开楼梯和台阶，优先使用电梯和坡道，并提示无障碍洗手间的位置。

**验收标准：**
- 无障碍模式开关明显可见
- 路线完全避开楼梯、台阶
- 显示沿途无障碍设施

### 2.5 导游 — 团队领队陈导

> 作为旅行团导游，我希望创建一个团队导航会话，团员扫码加入后可以看到我规划的参观路线，实时看到我的位置，掉队时可自动导航回团队。

**验收标准：**
- 导游创建房间，生成二维码
- 团员实时看到导游位置
- 掉队成员可一键导航回合

## 3. 核心功能

### 3.1 完整导航流程

```
搜索目的地 → 选择 POI → 显示路径预览 → 开始导航
→ 逐步指引（直行/左转/右转）→ 楼层切换提示（乘电梯到3F）
→ 到达目的地 → 导航结束
```

### 3.2 功能矩阵

| 功能 | 描述 | 优先级 |
|------|------|--------|
| POI 搜索 | 按名称、分类、关键词搜索目的地 | P0 |
| 路径规划 | A*/Dijkstra 最短路径计算 | P0 |
| 2D 地图导航 | 楼层平面图上绘制路径 | P0 |
| 多楼层切换 | 电梯/扶梯/楼梯衔接路由 | P0 |
| QR 码定位 | 扫描场景内二维码确定当前位置 | P0 |
| 多语言 UI | 中/英/日等语言切换 | P0 |
| 3D 路径渲染 | Three.js 数字孪生中动画路径 | P1 |
| BLE 实时定位 | 蓝牙信标连续定位 | P1 |
| 无障碍路由 | 轮椅可达路径规划 | P1 |
| 语音导航 | TTS 逐步语音播报 | P1 |
| 群组导航 | 导游模式 + 团员跟随 | P2 |
| 热力图 | 人流密度分析与拥堵规避 | P2 |
| 紧急疏散 | 动态疏散路径推送 | P2 |
| AI 推荐 | 基于偏好的 POI 推荐路线 | P3 |

## 4. MVP 范围与迭代计划

### Phase 1 — 基础导航（MVP）
**目标**：最小可用产品，验证核心价值  
**周期**：8 周

- 2D 楼层平面图渲染与切换
- A* 路径规划引擎
- QR 码 + 手动选择定位
- POI 管理后台（CRUD + 分类）
- 导航图可视化编辑器
- 基础多语言（中/英）
- 移动端 H5 + 触屏终端适配

### Phase 2 — 增强体验
**目标**：提升定位精度与沉浸感  
**周期**：6 周

- 3D Three.js 路径渲染（与数字孪生集成）
- BLE 信标实时定位
- WiFi 指纹定位（备选）
- 无障碍路由模式
- 语音导航（TTS）
- 多语言扩展（日/韩/马来等）
- 路径偏离检测与自动重新规划

### Phase 3 — 智能运营
**目标**：数据驱动的运营与安全能力  
**周期**：6 周

- 访客热力图实时分析
- 拥堵感知路径规划
- 群组导航（导游模式）
- 紧急疏散模式
- AI 智能推荐路线
- 与 SimPy 仿真引擎对接（疏散模拟）

## 5. 非功能需求

| 维度 | 要求 |
|------|------|
| **性能** | 路径规划 API 响应 < 500ms（1000 节点图） |
| **并发** | 支持 500 并发导航会话 |
| **可用性** | 核心导航功能离线可用（预缓存地图 + 本地路径计算） |
| **多语言** | 框架支持 i18n，MVP 中英双语，后续扩展 |
| **无障碍** | WCAG 2.1 AA 级，屏幕阅读器兼容 |
| **安全** | 定位数据匿名化，不采集个人信息 |
| **兼容性** | iOS Safari 15+, Android Chrome 90+, 主流触屏终端 |
| **部署** | 容器化部署，支持私有化与云端 |
| **数据** | 导航图数据支持增量更新，无需全量重载 |

## 7. 竞品分析

| 维度 | Mappedin | PointInside | Jibestream | **本方案** |
|------|----------|-------------|------------|-----------|
| 地图类型 | 2D/2.5D | 2D | 2D/3D | **2D + 3D 数字孪生** |
| 定位方式 | BLE | WiFi/BLE | BLE/UWB | **QR + BLE + WiFi** |
| 3D 支持 | 有限 | 无 | 基础 3D | **Three.js 完整 3D** |
| BIM 集成 | 无 | 无 | 有限 | **IFC/BIM 原生支持** |
| 无障碍路由 | ✓ | ✓ | ✓ | **✓** |
| 疏散模式 | ✗ | ✗ | 有限 | **✓ 动态疏散** |
| 热力图 | 付费 | ✓ | ✓ | **✓ 集成 DES 仿真** |
| 多语言 | ✓ | ✓ | ✓ | **✓** |
| 私有化部署 | ✗ | ✗ | ✓ | **✓** |
| 数字孪生联动 | ✗ | ✗ | ✗ | **✓ 原生集成** |

**核心差异化**：本方案基于 FactVerse 数字孪生平台，天然具备 BIM 数据集成、3D 场景渲染、DES 仿真分析能力，这是纯 SaaS 导航产品无法比拟的。

## 8. 风险与依赖

### 风险

| 风险 | 影响 | 概率 | 应对措施 |
|------|------|------|----------|
| BLE 信标部署成本高 | Phase 2 延迟 | 中 | MVP 使用 QR 码定位，降低硬件依赖 |
| 室内定位精度不足 | 用户体验差 | 中 | 多种定位融合 + 路径吸附算法 |
| 大型场馆节点数过多 | 路径计算超时 | 低 | 分层计算 + 预计算热门路径缓存 |
| 客户 BIM 数据质量差 | 导航图构建困难 | 中 | 提供导航图编辑器手动补充 |
| 多语言翻译质量 | 国际用户体验 | 低 | AI 翻译 + 人工审核流程 |

### 依赖

| 依赖项 | 提供方 | 状态 |
|--------|--------|------|
| Three.js 3D 引擎 | 基盘已有 | ✅ 可用 |
| BIM/IFC 导入 | web-ifc WASM | ✅ 可用 |
| 布局编辑器 | 基盘已有 | ✅ 可用 |
| SimPy 仿真引擎 | 基盘已有 | ✅ 可用 |
| BLE 信标硬件 | 第三方采购 | ⏳ Phase 2 需要 |
| TTS 语音服务 | 第三方 API | ⏳ Phase 2 需要 |
