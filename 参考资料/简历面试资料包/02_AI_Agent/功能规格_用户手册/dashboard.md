# Main Dashboard 用户指南

## Overview

主仪表盘是 FactVerse AI Agent 的核心入口页面，提供设施管理全局视图。用户登录后默认进入此页面，可一览关键绩效指标、告警趋势、各模块运行状态及 AI 智能洞察。

仪表盘设计遵循"一屏总览"原则，将最关键的运营数据浓缩在单个页面中，帮助设施管理者在 30 秒内掌握全局态势。

---

## KPI Cards

### 接口

```
GET /api/v1/dashboard/stats
```

### 说明

KPI 卡片区域位于仪表盘顶部，以数字卡片形式展示核心运营指标。每张卡片包含：

| 元素 | 说明 |
|------|------|
| 指标名称 | 如"活跃告警数"、"工单完成率" |
| 当前值 | 实时或近实时数值 |
| 趋势箭头 | 与上一周期对比的变化方向（↑/↓/→） |
| 变化百分比 | 环比变化率 |

### 响应示例

```json
{
  "activeAlerts": 12,
  "alertsTrend": -8.3,
  "workOrderCompletion": 94.2,
  "completionTrend": 2.1,
  "energyEfficiency": 87.5,
  "efficiencyTrend": 1.3,
  "occupancyRate": 72.0,
  "occupancyTrend": -0.5
}
```

### 使用说明

- 卡片支持点击跳转至对应模块详情页
- 数据默认每 5 分钟刷新一次，可通过右上角刷新按钮手动触发
- 趋势计算基于与前一个自然日/周/月的同期对比

## Alert Trend

### 接口

```
GET /api/v1/dashboard/alert-trend
```

### 说明

告警趋势图以折线图或面积图形式展示近期告警数量的变化趋势，帮助管理者识别告警高发时段和异常模式。

### 查询参数

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `period` | string | `7d` | 时间范围：`24h`、`7d`、`30d`、`90d` |
| `severity` | string | `all` | 告警级别过滤：`critical`、`warning`、`info`、`all` |
| `module` | string | `all` | 模块过滤：`smart-fm`、`traffic`、`heat`、`all` |

### 响应示例

```json
{
  "period": "7d",
  "dataPoints": [
    { "timestamp": "2026-02-11T00:00:00Z", "critical": 2, "warning": 8, "info": 15 },
    { "timestamp": "2026-02-12T00:00:00Z", "critical": 1, "warning": 6, "info": 12 }
  ],
  "summary": {
    "total": 187,
    "avgDaily": 26.7,
    "peakDay": "2026-02-14",
    "peakCount": 41
  }
}
```

### 使用说明

- 将鼠标悬停在图表上可查看具体时间点的告警分布
- 点击图例可切换显示/隐藏不同级别的告警曲线
- 支持框选放大查看特定时间段的详细数据

## Cross-Module Overview

跨模块总览区域以卡片形式展示三大核心模块的运行状态摘要：

### Smart FM（智慧设施管理）

| 指标 | 说明 |
|------|------|
| 设备在线率 | 受监控设备的在线比例 |
| 待处理工单 | 未关闭的维保工单数量 |
| 预测性维护提醒 | AI 预测即将需要维护的设备数 |
| 能耗评分 | 综合能效评分（0-100） |

### TrafficOps（交通运维）

| 指标 | 说明 |
|------|------|
| 当前人流量 | 实时客流统计 |
| 电梯/扶梯状态 | 运行中/停机/维护中的设备数 |
| 拥堵区域 | 当前检测到的拥堵热点数量 |
| 日均客流 | 当日累计客流与历史均值对比 |

### HeatOps（暖通运维）

| 指标 | 说明 |
|------|------|
| 平均温度 | 全楼宇加权平均温度 |
| 温控达标率 | 满足设定温度范围的区域占比 |
| 冷机运行状态 | 冷水机组的启停状态概览 |
| 节能建议 | AI 生成的当前可执行节能建议数 |

每张模块卡片右上角提供"查看详情"链接，点击可跳转至对应模块的完整管理页面。

## Notification Center

通知中心位于仪表盘右侧或顶部导航栏，集中展示系统内所有需要关注的事件。

### 通知类型

| 类型 | 图标 | 说明 |
|------|------|------|
| 告警通知 | 🔴 | 设备故障、阈值超限等紧急告警 |
| 工单通知 | 🟡 | 工单状态变更、到期提醒 |
| 系统通知 | 🔵 | 系统更新、维护窗口、配置变更 |
| AI 通知 | 🟢 | AI 分析结果、优化建议、异常检测 |

### 功能说明

- **未读标记**：未读通知以角标数字显示在通知铃铛图标上
- **快速操作**：部分通知支持直接在通知面板内执行操作（如确认告警、跳转工单）
- **过滤与搜索**：支持按类型、时间范围、模块来源过滤通知
- **批量操作**：支持全部标为已读、批量删除历史通知
- **通知偏好设置**：用户可自定义接收哪些类型的通知及推送方式（站内信/邮件/Webhook）

## AI Insights

AI 洞察面板展示由 AI 引擎自动分析生成的智能建议和发现，是仪表盘的核心差异化功能。

### 洞察类型

| 类型 | 说明 | 示例 |
|------|------|------|
| 异常检测 | 识别偏离正常模式的指标 | "3 楼 AHU-02 能耗较上周同期升高 23%，建议检查滤网" |
| 趋势预测 | 基于历史数据的趋势预判 | "按当前趋势，冷却塔 CT-01 预计 14 天后需维护" |
| 优化建议 | 可执行的运营优化方案 | "将 B 区温控设定从 23°C 调至 24°C，预计月省电 ¥1,200" |
| 关联发现 | 跨模块的关联性分析 | "周末客流下降 40% 时段冷机仍全负荷运行，建议联动调度" |

### 使用说明

- 每条洞察附带置信度评分（高/中/低），帮助用户判断优先级
- 支持"采纳"或"忽略"操作，用户反馈将持续优化 AI 模型
- 点击洞察卡片可展开查看详细分析数据和依据
- AI 洞察每小时自动刷新，也可手动触发重新分析
