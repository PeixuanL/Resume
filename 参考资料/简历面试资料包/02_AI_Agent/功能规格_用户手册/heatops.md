# HeatOps 用户指南

## 1. Overview — 概述

HeatOps 是 FactVerse 平台的智慧供热管理模块，专为集中供热企业的运行调度人员设计。模块覆盖供热全链路管理：

- **5 座换热站** 的实时监控与远程调节
- **管网拓扑** 可视化与泄漏检测
- **户控管理** — 逐户阀门控制与舒适度优化
- **AI 异常检测** 与负荷预测
- **MPC（模型预测控制）** — AI 生成最优调控策略

系统每 5 分钟通过仿真引擎更新全部运行数据，为调度决策提供实时支撑。

---

## 2. Overview Dashboard — 供热总览

查看全网运行概况，包括各站状态、总供热能力和系统效率。

**接口：** `GET /api/v1/heatops/overview`

**返回信息：**

- 各换热站运行状态（运行 / 停机 / 告警）
- 总供热容量（MW）与当前负荷
- 系统综合能效（COP）
- 室外温度与供热曲线偏差

**使用场景：** 每日交接班时快速掌握全网运行状况；大屏展示。

---

## 3. Heat Station Monitoring — 换热站监控

实时监控各换热站的运行参数，支持历史数据回溯。

**接口：**

| 操作 | 方法与路径 |
|------|-----------|
| 站点列表 | `GET /api/v1/heatops/stations` |
| 单站详情 | `GET /api/v1/heatops/stations/{id}` |
| 历史数据 | `GET /api/v1/heatops/stations/{id}/history` |

**监控参数：**

- **一次侧**：供水温度、回水温度、流量
- **二次侧**：供水温度、回水温度、流量
- **换热效率**（%）
- **阀门开度**（%）

**数据更新：** 仿真引擎每 5 分钟推送最新数据。历史查询支持按时间范围筛选。

**操作提示：**
- 当换热效率低于 85% 时，系统自动生成告警
- 阀门开度异常变化可能指示执行器故障

---

## 4. Pipe Network — 管网拓扑

可视化展示供热管网结构，支持泄漏检测与管段历史分析。

**接口：**

| 操作 | 方法与路径 |
|------|-----------|
| 管网拓扑 | `GET /api/v1/heatops/network/topology` |
| 泄漏检测 | `GET /api/v1/heatops/network/leaks` |
| 管段历史 | `GET /api/v1/heatops/network/history` |

**功能说明：**

- **拓扑可视化**：以图形方式展示管网连接关系、管径、长度、保温状况
- **泄漏检测**：基于压力和流量差异，AI 识别疑似泄漏点，标注位置与严重程度
- **管段历史**：查看指定管段的温度、压力、流量变化趋势

---

## 5. Household Control — 户控管理

逐户管理阀门开度和舒适模式，查看能耗统计，支持按楼栋聚合。

**接口：**

| 操作 | 方法与路径 |
|------|-----------|
| 户列表 | `GET /api/v1/heatops/households` |
| 阀门控制 | `POST /api/v1/heatops/households/{id}/valve-control` |
| 舒适模式 | `POST /api/v1/heatops/households/{id}/comfort-mode` |

**核心功能：**

- **阀门控制**：远程调节单户阀门开度（0–100%）
- **舒适模式**：设定目标室温，系统自动调节阀门以维持舒适度
- **舒适度评分**：综合室温、温度波动、响应速度等指标，0–100 分
- **能耗统计**：单户 / 楼栋维度的用热量、费用统计
- **楼栋聚合**：按楼栋查看整体供热质量与能耗分布

**操作提示：** 开启舒适模式后，手动阀门控制将被覆盖，系统自动托管调节。

---

## 6. Anomaly Detection — AI 异常检测

AI 自动识别供热系统中的异常状况。

**接口：** `GET /api/v1/heatops/anomalies`

**检测范围：**

- 换热站参数异常（温度突变、效率骤降）
- 管网压力 / 流量异常（疑似泄漏、堵塞）
- 户端温度偏差（供热不均、欠热投诉预警）
- 设备运行异常（泵、阀门、传感器故障）

**返回字段：** 异常类型、严重等级（低 / 中 / 高 / 紧急）、发生时间、关联设备、建议处置措施。

## 7. Load Forecast — 负荷预测

基于气象数据的供热负荷预测，辅助提前调度。

**接口：**

| 操作 | 方法与路径 |
|------|-----------|
| 负荷预测 | `GET /api/v1/heatops/forecast` |
| 气象数据 | `GET /api/v1/heatops/forecast/weather` |

**功能说明：**

- 结合未来 24–72 小时气象预报（温度、风速、日照），预测各时段供热需求
- 输出逐小时负荷曲线，标注峰值时段
- 支持与历史同期对比分析

**使用场景：** 调度员根据预测曲线提前调整锅炉出力和换热站阀门，避免供热不足或过量。

---

## 8. Model Predictive Control — MPC 策略

AI 生成全网最优调控策略，平衡供热质量与能耗成本。

**接口：** `POST /ai/mpc/strategy`

**策略内容：**

- 各换热站阀门开度建议值
- 循环泵频率调节建议
- 分时段供水温度曲线
- 预计节能效果（%）

**工作原理：** MPC 综合考虑负荷预测、管网热惯性、室外温度变化趋势，通过滚动优化计算未来数小时的最优控制序列。

**操作流程：** 生成策略 → 调度员审核 → 下发执行（参见"运行调度"章节）。

---

## 9. Operation Logs — 运行调度

调度指令的全生命周期管理，支持审批流程。

**接口：**

| 操作 | 方法与路径 |
|------|-----------|
| 调度记录列表 | `GET /api/v1/heatops/operation-logs` |
| 生成调度方案 | `POST /api/v1/heatops/operation-logs/generate/{buildingId}` |
| 审批 | `POST /api/v1/heatops/operation-logs/{id}/approve` |
| 执行 | `POST /api/v1/heatops/operation-logs/{id}/execute` |
| 取消 | `POST /api/v1/heatops/operation-logs/{id}/cancel` |

**审批流程：**

```
生成(generate) → 待审批(pending) → 审批通过(approve) → 执行(execute)
                                  ↘ 取消(cancel)
```

- **生成**：针对指定楼栋，由 AI 或调度员生成调度方案
- **审批**：主管审核方案合理性
- **执行**：下发控制指令到现场设备
- **取消**：在执行前可随时取消

所有操作记录可追溯，满足安全生产审计要求。

---

## 10. AI Suggestions — AI 建议

系统主动推送的智能优化建议。

**接口：** `GET /api/v1/heatops/suggestions`

**建议类型：**

- 能效优化（降低回水温度、调整运行模式）
- 设备维护提醒（基于运行时长和状态趋势）
- 供热质量改善（针对投诉热点区域）
- 调度策略建议（基于负荷预测的预调节）

**操作：** 每条建议支持 **采纳（accept）** 或 **驳回（reject）**。采纳后自动生成对应的调度指令进入审批流程。

---

## 11. Reports — 报告

自动生成运行报告，支持 JSON 数据接口和 PDF 下载。

**接口：**

| 报告类型 | 方法与路径 |
|---------|-----------|
| KPI 报告 | `GET /api/v1/heatops/reports/kpi` |
| 日报 | `GET /api/v1/heatops/reports/daily` |

**KPI 报告包含：**

- 系统综合能效、供热达标率、投诉率
- 各站运行时长与故障率
- 能耗趋势与同比 / 环比分析

**日报包含：**

- 当日各站运行参数汇总
- 异常事件与处理记录
- 调度指令执行情况
- 次日负荷预测与建议

两种报告均支持 JSON（用于系统集成）和 PDF（用于打印存档）格式。

---

## 12. Savings — 节能分析

按楼栋维度的节能效果与投资回报分析。

**接口：** `GET /api/v1/heatops/savings`

**分析内容：**

- 各楼栋节能量（kWh）与节能率（%）
- 节约费用（元）
- 改造投资回报率（ROI）
- 节能措施效果排名（户控改造、管网保温、智能调控等）

**使用场景：** 向管理层汇报智慧供热系统的投资回报，评估各项节能措施的实际效果，指导后续改造投资决策。
