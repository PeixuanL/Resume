# 告警管理用户指南

> 面向运维人员和管理人员的告警管理模块使用手册。

---

## 1. 概述

告警管理是 FactVerse 平台的核心模块，帮助运维团队实时掌握设备和系统异常状态，快速定位问题并采取行动。

**核心能力：**
- **智能告警** — AI 自动检测异常，支持 16 种检测场景
- **AI 归因分析** — 一键展开根因分析，了解告警来源和影响链
- **闭环处置** — 从告警到确认到解决，全流程可追溯
- **关联分析** — 自动识别相关告警，避免重复处理

**导航路径：** 侧边栏 → 告警管理

---

## 2. 告警列表

进入告警管理页面，可看到所有告警的列表视图。

### 筛选与排序

| 筛选条件 | 可选值 | 说明 |
|----------|--------|------|
| 状态 | 待处理 / 已确认 / 已解决 | 默认显示"待处理" |
| 严重程度 | 信息 / 低 / 警告 / 中 / 高 / 严重 | 6 级，可多选 |
| 告警类型 | 异常检测 / 阈值超限 / 预测预警 等 | 16 种类型 |

### 告警卡片信息

每条告警显示：
- **标题** — 告警简述（支持多语言显示）
- **严重程度** — 颜色标签（蓝=信息，黄=警告，红=严重）
- **关联设备** — 触发告警的设备名称，点击可跳转设备详情
- **触发时间** — 告警发生的时间
- **建议措施** — AI 推荐的处置方案

---

## 3. 告警详情

点击任意告警可展开详情面板。

### AI 归因分析面板

每条告警附带 AI 检测元数据，展开后可查看：

- **检测算法** — 使用的异常检测方法（如 Z-Score 统计检测、Isolation Forest 机器学习检测、AutoEncoder 深度学习重建误差等）
- **关键特征** — SHAP 值排序的影响因素，帮助理解"为什么触发了这条告警"
- **置信度** — 算法给出的可信度评分
- **影响范围** — 受影响的上下游设备
- **建议措施** — AI 推荐的处置方案和操作步骤

> **提示：** 归因面板因行业模块不同而展示不同内容。HeatOps 告警会显示换热站运行参数，SemiOps 告警会显示洁净室环境数据。

## 4. 告警处置流程

```
新告警（待处理）→ 确认（已确认）→ 解决（已解决）
```

| 操作 | 说明 | 权限要求 |
|------|------|----------|
| **确认** | 表示已知晓该告警，正在处理 | OPERATOR 或 ADMIN |
| **解决** | 标记告警已解决并记录处置结果 | OPERATOR 或 ADMIN |
| **创建工单** | 一键从告警创建维护工单 | OPERATOR 或 ADMIN |

> **注意：** 告警解决后即为最终状态，无法回退。

## 5. 告警类型说明

平台支持 16 种告警类型，按检测方式分为 4 大类：

### 异常检测类
- **通用异常** — AI 模型检测到的偏离正常模式的情况
- **统计异常** — 基于统计方法（Z-Score 等）的偏差检测
- **多信号关联** — 多个传感器信号同时异常的关联分析
- **振动-温度耦合** — 振动与温度的异常耦合关系

### 阈值类
- **阈值超限** — 传感器读数超过预设阈值
- **ISO 标准突破** — 超过 ISO 国际标准规定的限值

### 预测类
- **预测预警** — AI 预测即将发生的异常
- **预测性维护** — 基于设备健康评分的维护预警
- **健康指数警告/危险** — 设备健康评分降至警戒/危险线

### 趋势类
- **数据漂移** — 传感器数据逐渐偏离基线
- **趋势变化** — 数据呈现新趋势
- **趋势加速** — 恶化趋势加速
- **灾难性故障** — 预测到严重故障风险

## 6. 常见问题

**Q: 告警太多，如何优先处理？**
建议按严重程度排序，优先处理 HIGH 和 CRITICAL。同时关注关联告警数量——关联多条告警的事件往往是系统性问题。

**Q: 告警和工单是什么关系？**
告警是系统自动检测到的异常信号。您可以从告警一键创建工单，分配给维护人员执行。一条告警可以关联一个工单。

**Q: AI 告警的置信度如何理解？**
置信度 0-1 之间。0.8 以上通常表示高可信度，建议优先处理。0.5-0.8 建议人工复核。低于 0.5 可能是噪音。
