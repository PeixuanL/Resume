## 3. AI 顾问 | AI Advisor

📍 **在任意页面可用** — 点击右下角的 🤖 **浮动按钮**

**怎么用：**

1. 点击任意页面右下角的 🤖 按钮，打开 AI 对话面板
2. 输入自然语言问题，AI 顾问会调用 **60 个内置工具** 为你分析、查询、模拟

**试试这些问题：**

| 你问 | AI 做什么 |
|------|----------|
| `Chiller #3 什么状态？` | 查询设备实时状态、传感器数据 |
| `最近有什么告警？` | 汇总当前活跃告警 |
| `run a simulation` | 启动离散事件仿真（DES） |
| `帮我算一下 ROI` | 调用价值量化引擎 |
| `交通流量预测` | 调用 TrafficOps 模块 |
| `生成一份报告` | 汇总数据生成执行报告 |

**How to use it:**

1. Click the 🤖 **floating button** in the bottom-right corner of any page
2. Type natural language questions — the AI Advisor uses **60 built-in tools** to analyze, query, and simulate

**Try these prompts:**

| You ask | AI does |
|---------|---------|
| `Chiller #3 什么状态？` | Queries real-time equipment status & sensor data |
| `What alerts are active?` | Summarizes current active alerts |
| `run a simulation` | Launches a DES simulation |
| `Calculate ROI` | Runs the value quantification engine |
| `Traffic forecast` | Invokes the TrafficOps module |
| `Generate a report` | Compiles data into an executive report |

**API 调用 | API Call：**

```bash
# 与 AI 顾问对话 | Chat with AI Advisor
curl -X POST https://factverse.ai/api/ai/chat \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"message": "Chiller #3 什么状态？"}'
```

## 4. 告警管理 | Alerts

📍 **URL:** [https://factverse.ai/alerts](https://factverse.ai/alerts)

**演示环境包含 5 条告警：**

| 级别 | 数量 | 示例 |
|------|------|------|
| 🔴 CRITICAL | 2 | 关键设备故障告警 |
| 🟡 WARNING | 2 | 性能降级 / 阈值预警 |
| 🟢 ACKNOWLEDGED | 1 | 已确认并处理中 |

**操作步骤：**

1. 左侧导航点击 **告警管理** 或直接访问上方 URL
2. 看到告警列表，按严重级别排列
3. **点击任意告警** → 展开详情面板
4. 查看 **RCFA（根因故障分析）** — AI 自动生成的根因分析报告
5. 可点击"确认"按钮将告警标记为已处理

**The demo has 5 alerts:**

| Severity | Count | Example |
|----------|-------|---------|
| 🔴 CRITICAL | 2 | Critical equipment failure |
| 🟡 WARNING | 2 | Performance degradation / threshold warnings |
| 🟢 ACKNOWLEDGED | 1 | Confirmed and being handled |

**Steps:**

1. Click **Alerts** in the left nav or go to the URL above
2. See the alert list sorted by severity
3. **Click any alert** → detail panel opens
4. View the **RCFA (Root Cause Failure Analysis)** — auto-generated by AI
5. Click "Acknowledge" to mark as handled

**API 调用 | API Call：**

```bash
# 获取告警列表 | Get alerts
curl -H "Authorization: Bearer $TOKEN" \
  https://factverse.ai/api/alerts

# 获取单条告警详情 + RCFA | Get alert detail with RCFA
curl -H "Authorization: Bearer $TOKEN" \
  https://factverse.ai/api/alerts/{alert_id}
```

## 5. 仿真引擎 | Simulation

📍 **URL:** [https://factverse.ai/simulation](https://factverse.ai/simulation)

**操作步骤：**

1. 左侧导航点击 **仿真引擎** 或直接访问上方 URL
2. 选择仿真场景（演示环境已预设场景）
3. 点击 **▶ 运行仿真** 按钮
4. 等待 DES（离散事件仿真）引擎计算完成
5. 查看仿真结果：
   - 📊 时序图 — 设备负载随时间变化
   - 📈 对比分析 — 优化前 vs 优化后
   - 📋 建议清单 — AI 生成的优化建议

**Steps:**

1. Click **Simulation** in the left nav or go to the URL above
2. Select a simulation scenario (demo has presets)
3. Click **▶ Run Simulation**
4. Wait for the DES (Discrete Event Simulation) engine to complete
5. View results:
   - 📊 Time-series chart — equipment load over time
   - 📈 Before vs. after comparison
   - 📋 Recommendation list — AI-generated optimization suggestions

> 💡 也可以在 AI 顾问中输入 `run a simulation` 直接触发。
>
> 💡 You can also type `run a simulation` in the AI Advisor to trigger it directly.

**API 调用 | API Call：**

```bash
# 运行仿真 | Run simulation
curl -X POST https://factverse.ai/api/simulation/run \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"scenario": "default"}'

# 获取仿真结果 | Get simulation results
curl -H "Authorization: Bearer $TOKEN" \
  https://factverse.ai/api/simulation/results/{run_id}
```

## 9. 暗色模式 | Dark Mode

📍 **设置入口：** 点击右上角头像 → **设置** → **外观** → 切换 **暗色模式**

**Steps:**

1. 点击页面右上角的用户头像
2. 选择 **设置 / Settings**
3. 找到 **外观 / Appearance** 选项
4. 切换 **暗色模式 / Dark Mode** 开关
5. 界面即时切换为暗色主题 🌙

**Steps (English):**

1. Click the user avatar in the top-right corner
2. Select **Settings**
3. Find **Appearance**
4. Toggle **Dark Mode**
5. The interface switches to dark theme instantly 🌙
