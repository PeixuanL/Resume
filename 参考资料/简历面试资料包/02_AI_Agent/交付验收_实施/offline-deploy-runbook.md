# Offline Deployment Runbook

> For deployments at customer sites with no internet access.
> Package generated by: `offline-deploy-package.yml` CI workflow.

---

## Prerequisites

- Docker Engine 24.x+ installed on target machine
- Minimum 50 GB free disk space
- Root/sudo access
- USB drive or internal artifact server (to transfer package)

---

## Step 1: Verify Package Integrity

On the target machine:

```bash
# Verify checksums
sha256sum -c CHECKSUMS.txt

# All files should show OK. If any show FAILED, the package is corrupted.
# Do NOT proceed if checksums fail.
```

---

## Step 2: Load Docker Images

```bash
# Load all images (run from the package directory)
gunzip -c images/factverse-backend-latest.tar.gz | sudo docker load
gunzip -c images/factverse-frontend-latest.tar.gz | sudo docker load
gunzip -c images/factverse-ai-engine-latest.tar.gz | sudo docker load

# Verify loaded images
sudo docker images | grep factverse
```

Expected output:
```
factverse-backend     latest
factverse-frontend    latest
factverse-ai-engine   latest
```

---

## Step 3: Configure Environment

```bash
# Copy templates and edit
cp configs/backend.env.example /data/apps/factverse-backend.env
cp configs/ai-engine.env.example /data/apps/factverse-ai-engine.env

# Edit with your environment values
sudo vi /data/apps/factverse-backend.env
sudo vi /data/apps/factverse-ai-engine.env
```

Required variables:

**backend.env:**
```bash
DB_HOST=fv-postgres
DB_USER=postgres
DB_PASSWORD=<your-password>
REDIS_HOST=fv-redis
REDIS_PASSWORD=<your-password>
SPRING_PROFILES_ACTIVE=prod
CORS_ALLOWED_ORIGINS=http://localhost:3001
SERVICE_PASSWORD=<admin-password>
```

**ai-engine.env:**
```bash
DB_HOST=fv-postgres
DEMO_MODE=false          # Set to false only if local LLM is configured
# Add local LLM endpoint if DEMO_MODE=false
# LLM_ENDPOINT=http://localhost:11434
```

---

## Step 4: Start Database and Redis

```bash
# Start PostgreSQL
sudo docker run -d \
  --name fv-postgres \
  -p 5432:5432 \
  -e POSTGRES_DB=factverse \
  -e POSTGRES_USER=postgres \
  -e POSTGRES_PASSWORD=<your-password> \
  -v /data/postgres:/var/lib/postgresql/data \
  timescale/timescaledb:latest-pg16

# Start Redis
sudo docker run -d \
  --name fv-redis \
  -p 6379:6379 \
  -e REDIS_PASSWORD=<your-password> \
  -v /data/redis:/data \
  redis:7-alpine --requirepass <your-password>
```

Wait 30 seconds for startup, then verify:
```bash
sudo docker ps
curl -s http://localhost:5432  # Should return connection refused or similar (postgres doesn't have HTTP)
psql -U postgres -h localhost -c "SELECT 1"  # Should succeed
```

---

## Step 5: Run Database Migrations

```bash
# Apply all Flyway migrations
flyway -url=jdbc:postgresql://localhost:5432/factverse \
       -user=postgres \
       -password=<your-password> \
       -locations=filesystem:/migrations \
       migrate

# Verify migration count
flyway -url=jdbc:postgresql://localhost:5432/factverse \
       -user=postgres \
       -password=<your-password> \
       info
```

---

## Step 6: Deploy Application Components

```bash
# Deploy all three components
sudo bash scripts/deploy-all-3.sh

# OR deploy individually:
sudo bash scripts/deploy-fe.sh
sudo bash scripts/blue-green-be.sh
```

Expected startup time:
- Backend (Spring Boot): ~60 seconds
- Frontend (Vite): ~10 seconds
- AI Engine (FastAPI): ~20 seconds

---

## Step 7: Post-Deploy Verification

```bash
# Run health check
bash scripts/healthcheck.sh

# Run offline verification
BASE_URL=http://localhost:8080 AI_URL=http://localhost:8000 \
  bash scripts/offline-deploy-check.sh
```

Required checks:
- [ ] Backend `/api/v1/health` → HTTP 200
- [ ] AI Engine `/health` → HTTP 200
- [ ] Frontend → HTTP 200 (http://localhost:3001)
- [ ] No external network calls (verify with `sudo netstat -tpn`)

## Offline-Specific Checks

### 1. Verify no external CDN dependencies
```bash
# Check if any containers are trying to reach external networks
sudo docker logs fv-frontend 2>&1 | grep -i "cdn\|unpkg\|jsdelivr" || echo "OK"
sudo docker logs fv-backend 2>&1 | grep -i "connection refused\|timeout" || echo "OK"
```

### 2. Verify DEMO_MODE for AI Engine
```bash
sudo docker exec fv-ai-engine env | grep DEMO_MODE
# Expected: DEMO_MODE=true
# If false, ensure local LLM is configured and reachable
```

### 3. Verify volume persistence
```bash
sudo docker inspect fv-postgres --format '{{.Mounts}}'
# Should show /data/postgres mounted
```

## Troubleshooting

### Backend won't start
```bash
# Check logs
sudo docker logs fv-backend --tail 100

# Common causes:
# - DB password wrong → check backend.env
# - DB not ready → wait 60s after DB start
# - Port 8080 occupied → sudo lsof -i :8080
```

### AI Engine returns 500 on /ai/advisor/chat
```bash
# Check if DEMO_MODE=true (required for offline)
sudo docker exec fv-ai-engine env | grep DEMO_MODE

# If DEMO_MODE=false, verify LLM endpoint is reachable inside container
sudo docker exec fv-ai-engine curl -s http://localhost:11434/v1/models
```

### Frontend shows blank page
```bash
# Check browser console for network errors
# Verify API base URL is correct
sudo docker exec fv-frontend env | grep VITE_API
```

## Data Backup & Restore

```bash
# Create snapshot before upgrade
bash scripts/snapshot.sh save pre-upgrade

# Restore snapshot
bash scripts/snapshot.sh restore pre-upgrade

# List available snapshots
bash scripts/snapshot.sh list
```

---

## Version Upgrade Procedure

1. Load new Docker images (Step 2)
2. Backup current database (Step 5 / backup.sh)
3. Take pre-upgrade snapshot: `bash scripts/snapshot.sh save pre-upgrade-v{new}`
4. Apply new migrations: `flyway migrate`
5. Restart containers: `docker-compose down && docker-compose up -d`
6. Run post-deploy verification (Step 7)

---

## Emergency Rollback

```bash
# Stop current containers
sudo docker stop fv-backend fv-frontend fv-ai-engine
sudo docker rm fv-backend fv-frontend fv-ai-engine

# Restore previous images (requires previous image to be saved)
gunzip -c images/factverse-backend-{old-tag}.tar.gz | sudo docker load
gunzip -c images/factverse-frontend-{old-tag}.tar.gz | sudo docker load
gunzip -c images/factverse-ai-engine-{old-tag}.tar.gz | sudo docker load

# Restore DB from snapshot
bash scripts/snapshot.sh restore pre-upgrade

# Restart with old images
# (edit deploy scripts to use {old-tag} image names)
```
