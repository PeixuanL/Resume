
### 场景名称：智能体协同下的循环泵轴承隐患“无感”消除

#### 1. 场景基本设定
*   **对象**：2号楼机房 - #02 冷却循环泵（关键设备）。
*   **故障模式**：驱动端轴承早期磨损（尚未停机，但振动异常）。
*   **涉及角色**：
    *   **AI Agents**：告警Agent、检索Agent、工单Agent。
    *   **人类**：运维经理（决策者）、技师老张（执行者）。

---

### 2. 详细业务流程拆解 (Step-by-Step)

这个流程展示了从**AI感知**到**AI执行**的全过程，每一个步骤都对应一个具体的Agent动作。

#### 步骤一：隐患捕获与清洗 (告警 Agent)
> **任务目标**：将“数据波动”转化为“有效风险事件”，过滤误报。

*   **输入**：预测算法监测到 #02泵 的振动加速度峰值出现间歇性升高，算法预测 RUL（剩余寿命）为 72小时。
*   **告警 Agent 动作**：
    1.  **状态校验**：调用设备运行状态，确认当前泵处于“平稳运行”状态，而非“启停过程”（排除启停冲击造成的误报）。
    2.  **趋势比对**：检索该设备过去3个月的基准线，确认当前振动值偏离正常区间 15%。
    3.  **定级**：判定为“高置信度预测性告警”，而非“紧急停机故障”。
*   **输出**：生成一个内部事件ID `Evt-20240305-02`，并唤醒 **检索 Agent**。

#### 步骤二：情报收集与诊断 (检索 Agent)
> **任务目标**：像专家一样查阅资料，为维修做准备，而不是仅仅扔出一个报警代码。

*   **输入**：事件ID `Evt-20240305-02`，故障类型：轴承磨损。
*   **检索 Agent 动作**：
    1.  **查台账**：锁定 #02泵 型号为 *Grundfos NK 100-200*。
    2.  **查BOM（物料清单）**：定位驱动端轴承型号为 *SKF 6309-2RS*。
    3.  **查库存**：调用ERP接口，确认 *SKF 6309-2RS* 当前仓库库存 **3个**，位置在 **A区-05货架**。
    4.  **查知识库**：提取该型号泵的《轴承更换SOP》，并标记关键扭矩参数“螺栓紧固力矩 45Nm”。
*   **输出**：生成一份包含备件信息、库存位置、作业指导书的“诊断包”，转交给 **工单 Agent**。

#### 步骤三：资源调度与方案生成 (工单 Agent)
> **任务目标**：计算“什么时间修”对业务影响最小，且“谁去修”最合适。

*   **输入**：诊断包 + 设备ID。
*   **工单 Agent 动作**：
    1.  **查排程**：扫描设备运行计划，发现 #02泵 在 **明天（周三）上午 10:00 - 12:00** 为备用轮转状态（停机窗口期），维修不影响业务。
    2.  **查人员**：扫描排班表，发现高级技师“老张”在明早 10:00 有空闲，且老张的历史技能标签中有“水泵维修专家”。
    3.  **预填单**：草拟工单，填入上述所有信息（时间、人、备件、SOP）。
*   **输出**：构建一个待审批的“完美维修提案”。

#### 步骤四：人类决策 (Dashboard 呈现) -> **演示的高潮点**
> **任务目标**：用户只需要做“选择题”，而不是“问答题”。

*   **界面表现**：
    运维经理的 Dashboard 弹出一个 **“AI 助理通知”** 窗口，内容如下：
    > **🔴 预测性维护建议：#02 循环泵**
    > **诊断**：检测到轴承磨损风险，预计72小时后可能故障。
    > **AI 方案**：
    > 1. **时间**：建议预约 **明天 10:00**（设备轮休期）。
    > 2. **备件**：型号 6309-2RS（库存充足，已锁定）。
    > 3. **人员**：指派给 **技师老张**。
    > **附件**：振动频谱图.png, 维修SOP.pdf
    >
    > [ 批准执行 ]  [ 修改方案 ]  [ 忽略 ]

*   **用户动作**：经理点击 **[ 批准执行 ]**。

#### 步骤五：自动执行与闭环 (工单 Agent & 算法)
> **任务目标**：任务分发与结果验证。

*   **工单 Agent 动作**：
    1.  正式生成工单 `WO-20240305-88`。
    2.  通过 App 向技师老张发送推送：“明天10点维修#02泵，备件在A区05架，请按SOP执行。”
*   **技师动作（线下）**：老张第二天按时维修，更换轴承，在App点击“完成”。
*   **闭环验证**：
    1.  设备重新上电。
    2.  **算法**再次介入，监测运行数据 15分钟。
    3.  确认振动值恢复绿区，健康度评分从 60分 回升至 98分。
    4.  工单自动归档，并在Dashboard上标记为“成功避免一次潜在停机”。

### 4. 场景亮点总结 (用于文档)

这个场景虽然只涉及换一个轴承，但它完整闭环了：
*   **感知层**：算法 + 告警Agent（去伪存真）。
*   **认知层**：检索Agent（关联数据，打破孤岛）。
*   **决策层**：工单Agent（优化资源配置）。
*   **执行层**：App推送 + 闭环验证。

这就是一个极具说服力的、基于AI Agent的现代化运维场景。
