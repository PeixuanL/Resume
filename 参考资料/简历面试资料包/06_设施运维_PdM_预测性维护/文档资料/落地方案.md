### 二、 触发方式 (Trigger Mechanisms)
*(保持与上一版核心逻辑一致，确保代码生成的事件监听明确)*

1. **阶段一：告警生成（系统级自动触发）**
   * **触发逻辑**：算法系统/规则引擎判定 `Confidence > 0.8 && RUL < 72h`。
   * **动作**：通过 WebSocket/SSE 向**监控看板模块**推送全局告警事件 `Alarm_Triggered`。
2. **阶段二：诊断Agent唤醒（用户交互触发）**
   * **触发逻辑**：用户点击告警列表或拓扑图中的异常设备卡片。
   * **动作**：唤醒“诊断Agent”，调用时序数据模块与台账模块，开始根因分析。
3. **阶段三：调度Agent唤醒（用户交互触发）**
   * **触发逻辑**：用户在诊断报告页点击【制定维修方案】。
   * **动作**：唤醒“调度Agent”，并发调用库存模块、排班模块及动环负载模块接口。
4. **阶段四：生成Agent唤醒（用户交互触发）**
   * **触发逻辑**：用户确认调度资源后，点击【草拟工单】。
   * **动作**：唤醒“生成Agent”，调用 RAG 知识检索模块与工单草稿创建接口。
5. **阶段五：审批下发（用户决策触发）**
   * **触发逻辑**：用户在工单模块点击【批准并派发】。
   * **动作**：调用工单模块的发布接口，流转至执行端（技师App）。

### 三、 每一环节需要调用的 API (API Contracts)

这里为你整理出标准化的 API 契约，**你可以直接将这部分提供给 Vibe Coding 工具（如 Cursor/Windsurf），让其自动生成前后端对接代码（Axios 请求或 Fetch API）：**

#### 1. 设备诊断阶段 (Agent 1)
*   **读取时序数据**: `GET /api/v1/device/{device_id}/telemetry`
    *   *Params*: `metrics=vibration,current`, `time_range=last_24h`
*   **获取历史工单**: `GET /api/v1/maintenance/history`
    *   *Params*: `device_id=AHU-02`, `limit=5`
*   **知识库RAG检索**: `POST /api/v1/knowledge/rag_search`
    *   *Payload*: `{"query": "AHU精密空调皮带打滑特征及张力校准标准", "top_k": 3}`

#### 2. 工单调度阶段 (Agent 2)
*   **查询备件库存**: `GET /api/v1/inventory/stock`
    *   *Params*: `part_model=B-52`, `warehouse_id=DC_WH_01`
*   **锁定备件**: `POST /api/v1/inventory/reserve` (重要：防并发超卖)
    *   *Payload*: `{"part_id": "P-B52-001", "qty": 2, "reserve_time": "24h"}`
*   **查询技师排班**: `GET /api/v1/staff/schedule`
    *   *Params*: `skill_required=HVAC`, `date=tomorrow`
*   **获取机房负载**: `GET /api/v1/datacenter/load_forecast`
    *   *Params*: `zone_id=Room_02`, `date=tomorrow`

#### 3. 工单生成阶段 (Agent 3)
*   **获取工单模板**: `GET /api/v1/workorder/templates`
    *   *Params*: `type=mechanical_repair`
*   **提取标准SOP**: `POST /api/v1/knowledge/extract_sop`
    *   *Payload*: `{"task": "更换精密空调风机皮带并校准张力"}`
*   **生成工单草稿**: `POST /api/v1/workorder/draft`
    *   *Payload*: `{完整的JSON结构，包含设备、描述、时间、执行人、备件、SOP等}`

#### 4. 审批派发阶段 (最后一步)
*   **发布工单**: `POST /api/v1/workorder/{draft_id}/publish`
    *   *Response*: `{"status": "success", "workorder_id": "WO-20231024-001"}`

### 四、 产品最终呈现方式：标准化产品功能模块架构

在设计这个“人机协同预测性维护”产品时，它不再是一个单一的页面，而是一个包含多个标准功能模块的**系统级应用 (Predictive Maintenance Copilot System)**。最终呈现的标准化产品包含以下四大核心功能模块：

#### 模块一：智能全景监控大屏 (Smart Monitoring Dashboard)
*定位：发现问题的“眼睛”，承载阶段一的告警触发功能。*
* **核心功能点**：
  * **3D/2D 机房拓扑视图**：可视化展示数据中心各设备节点（AHU精密空调等）的实时运行状态。
  * **全局告警中心 (Alert Center)**：集中呈现来自底层算法的预测性告警（如：AHU-02 皮带打滑预警，倒计时 72h）。
  * **健康度指数 (Health Score)**：通过仪表盘直观展示当前机房的整体健康度。
* **Vibe Coding 呈现效果**：屏幕中央是设备拓扑，AHU-02 节点闪烁黄灯。右侧为滚动告警列表，用户点击告警条目即可进入“设备健康诊断模块”。

#### 模块二：设备健康与诊断工作台 (Health & Diagnostic Workspace)
*定位：分析问题的“大脑”，内嵌 Agent 1（设备诊断 Analyst）。*
* **核心功能点**：
  * **时序数据可视化看板**：联动遥测模块，将干瘪的数据转化为可视化的振动频谱图（FFT）和趋势曲线，高亮异常频段（如12Hz/25Hz）。
  * **AI 诊断报告区**：
    * 自动输出设备当前病症（如：皮带松弛）。
    * 自动关联展示历史维修台账（如：上次更换是12个月前）。
  * **行动建议触发器**：提供明确的决策按钮（`[让AI制定维修方案]`），引导用户进入下一环节。
* **Vibe Coding 呈现效果**：左侧展示动态图表和设备档案，右侧是一个类似 Copilot 的对话/流式输出面板，以打字机效果呈现 Agent 的诊断推演过程。

#### 模块三：AI 协同调度中心 (AI Dispatch Center)
*定位：解决问题的“大管家”，内嵌 Agent 2（工单调度 Coordinator）。*
* **核心功能点**：
  * **多维资源匹配引擎**：在后台并发核对“备件库存”、“人员排班”、“机房低负载时间窗口”。
  * **最优方案比选卡片**：将匹配结果以“卡片”形式呈现。
    * *例如*：展示“备件 B-52（库存4件，已预占2件）”、“人员：王工（具备HVAC资质）”、“时间：明日10:00-11:00（预测负载最低）”。
  * **方案干预与确认机制**：允许运维经理在卡片上微调方案（例如手动将王工换成李工），确认无误后点击 `[一键草拟工单]`。
* **Vibe Coding 呈现效果**：界面以 Kanban（看板）或资源卡片的形式展示。Agent 完成查询后，高亮显示匹配成功的资源，并提供修改下拉框供人类干预。

#### 模块四：智能工单流转模块 (Smart Work Order Management)
*定位：落地执行的“手脚”，内嵌 Agent 3（工单生成 Scribe）。*
* **核心功能点**：
  * **工单草稿生成器**：根据前三个模块的上下文，自动将故障描述、调度信息填入工单表单。
  * **SOP与安全规范自动注入**：联动知识库 RAG 模块，自动提取标准作业程序（如：断电挂牌步骤）并作为必读附件/检查项挂载到工单中。
  * **审批与派发流转 (Workflow)**：标准的工单状态机（Draft -> Approved -> In Progress -> Resolved）。运维经理只需进行最后的审批确认。
* **Vibe Coding 呈现效果**：居中弹出一个标准的“工单详情表单（Form）”。关键字段（SOP、物料、人员）均由 AI 预填并打上“AI 生成”的角标。底部是粗体的 `[批准并派发]` 按钮。点击后状态流转，工单下发至移动端。

### 总结：如何用这套方案进行 Vibe Coding？

作为 PM，你可以用以下结构指导研发（或 AI 编程助手）进行产品拼装：

> **“我们要构建一个预测性维护产品。请先搭建整体布局：包含【监控大屏模块】、【诊断工作台模块】、【调度中心模块】和【工单流转模块】的路由结构。**
> 
> **第一步**，在【监控模块】中，监听 `IoT遥测模块` 和 `模型推理模块` 的数据，实现设备告警的 UI 触发。
> **第二步**，当进入【诊断模块】时，通过 Context 传递 `device_id`，调用 `Agent 1` 的接口，在右侧面板流式展示基于历史台账的诊断结果。
> **第三步**，在【调度模块】中，利用 `Agent 2` 并发请求 `ERP库存模块` 和 `HR排班模块`，渲染资源推荐卡片。
> **最后**，在【工单模块】中，利用 `Agent 3` 调用 `知识库模块` 提取 SOP，渲染出完整的工单表单，并接入后端的审批派发接口完成闭环。”
