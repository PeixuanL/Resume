# 1. 需求来源
- 本次需求来源于Ameco项目的成功交付经验总结与标品化沉淀。
- 在标准产品旧版本的数据探索工作流中，长期存在以下痛点，阻碍了产品体验与系统稳定性： 
	- **用户体验差（交互冗杂）：** 在分析多个参数时，用户必须对节点中的每一个参数进行单独的连线操作。这种“蜘蛛网式”的连线方式带来了极大的维护负担，导致工作流复用性差、扩展效率极低。 
	- **技术架构脆弱（维护成本高）：** 基于参数级的连线逻辑，导致后台程序在解析和生成表达式时异常复杂。底层逻辑高度耦合，任何微小的变动都可能导致大面积的故障（故障影响面大），极易产生 BUG，限制了产品的进一步迭代。 
- 基于上述痛点，结合定制项目中的成功验证，我们需要将优化后的“节点级连线+内部参数配置”交互模式合并至标准产品线。
# 2. 产品功能价值 
* **用户价值（降本增效）：** 
	* **直观易用：** 采用标准的“输入 - 处理 - 输出”节点结构，工作流处理逻辑一目了然，大幅降低新用户的学习成本。 
	* **效率提升：** 将参数连线改为在节点内部配置，释放了画布的视觉空间，消除了繁琐的连线操作，提升了数据分析师搭建和修改工作流的效率。 
* **技术价值（解耦与健壮性）：** 
	* **架构解耦：** 将原本复杂的“参数级连接”简化为“节点级链接”，大幅降低后台解析引擎的压力与复杂度。 
	* **提升系统稳定性：** 降低逻辑耦合度，缩小故障影响半径，从根本上减少工作流执行和解析过程中的 BUG 产出率。

# 3. 功能目标
* **交互模式重构：** 废除旧版“参数连线”机制，全面升级为“节点连线”。节点间的连接仅代表数据流向，具体的参数映射关系移至节点属性面板中进行配置。

##### B. 页面数据、操作及按钮说明
1. 页面操作
	1. 本版本优化内容，主要是中间区域（任务编排区域）的优化
		* **添加节点：** 从左侧节点组件栏长按并拖拽目标节点至画布区的空白处，完成节点创建。 
		* **连接节点（定义流向）：** 拖拽节点边缘的点连线锚点至下游节点完成连接。连线用于定义上下游的执行依赖与数据整体流向。 
		* **配置节点：** 双击选定画布中的节点，在唤起的右侧节点配置面板中进行配置。配置区包含以下三个部分：
			* **输入配置：** 选择由上游连线节点传入的参数，同时支持对参数进行重命名。 （参数只可选择一次，已选参数置灰）
			* **输出配置：** 定义并确认当前节点处理完成后，向下一级节点输出的参数名称。（每一节点的输出配置不同，由6.2所述）
	2.  其余交互同8.0版本的DFS。
2. 页面字段说明
数据探索画布编排部分的页面字段与DFS8.0相同。仅修改了连线部分的交互，因此本文档不再赘述页面字段说明。
##### C. 异常情况处理方案
###### 节点与连线交互异常
| 异常场景          | 触发条件                                              | 处理方案及交互说明                                                                            |
| :------------ | :------------------------------------------------ | :----------------------------------------------------------------------------------- |
| **闭环/死循环连线**  | 用户尝试将下游节点的输出端口连线至其上游节点的输入端口（例如 A -> B -> C -> A）。 | **强阻断：** 连线自动吸附失败并断开。出现全局 Toast 提示：`“不支持循环连线，请重新规划数据流向。”`                            |
| **重复连线**      | 用户尝试在已经存在连线的两个节点之间，再次建立相同方向的连线。                   | **静默处理：** 连线失败，松手后自动撤销该连线动作，保持原有单条连线状态，不额外报错。                                        |
| **删除带有依赖的连线** | 用户选中并删除了节点A至节点B的连线，但节点B内部已配置了来源于节点A的输入参数。         | **状态置为失效：** 连线删除成功。<br>1.  打开节点B配置面板，原先引用的节点A输入参数标红展示，并提示`“来源节点已断开，参数失效”`。           |
| **删除带有依赖的节点** | 用户删除了一个有下游节点的节点，且下游节点引用了该被删节点的输出参数。               | **级联失效：** 节点及相关连线删除成功。<br>下游节点在画布上亮起异常状态图标。下游配置面板内涉及的输入参数清空来源或标红提示`“来源节点已删除，请重新配置”`。 |

##### C. 异常情况处理方案
###### 1. 参数配置与校验异常
| 异常场景           | 触发条件                                           | 处理方案及交互说明                                                                          |
| :------------- | :--------------------------------------------- | :--------------------------------------------------------------------------------- |
| **节点重命名冲突**    | 用户在“节点配置/重命名”弹窗中，输入的名称与画布中已有节点名称完全一致。          | **表单校验（强阻断）：** 输入框下方出现红色提示文案：`“当前画布中已存在同名节点，请修改”`。此时【确定】按钮置灰不可点击。                  |
| **输出参数命名不合规**  | （结合文档字段说明）用户自定义的输出参数名包含特殊字符，或以数字开头。            | **表单校验（强阻断）：** 实时校验，输入框下方标红提示：`“只能包含字母、数字、空格或下划线，并且以字母或下划线开头”`。未修改正确前，阻止当前节点的配置保存。 |
| **输出参数命名冲突**   | 用户在同一节点内，将两个不同的输出参数重命名为相同的名称。                  | **表单校验（强阻断）：** 标红冲突的输入框，提示`“输出参数名称不可重复”`，阻止保存。                                     |
| **必填项未配置（空置）** | （如运算节点、特征提取节点的输出参数设定了必填）用户未填写输出参数直接点击空白处或关闭面板。 | 未填写节点展示输出字段为【未定义】标红                                                                |

###### 2. 上下游数据流转（解耦映射）异常

| 异常场景           | 触发条件                                                         | 处理方案及交互说明                                                     |
| :------------- | :----------------------------------------------------------- | :------------------------------------------------------------ |
| **上游输出参数被重命名** | 节点A的输出参数“Field_1”被重命名为“Field_2”，而下游节点B已经配置了输入参数来源于“Field_1”。 | **自动同步：** 系统自动识别映射关系，将下游节点B中的输入参数源自动更新为“Field_2”，保障数据流不断裂。    |
