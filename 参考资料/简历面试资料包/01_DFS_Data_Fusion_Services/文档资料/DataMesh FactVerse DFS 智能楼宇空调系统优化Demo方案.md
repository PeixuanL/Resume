

## 1. 方案概述

### 1.1 演示目标

通过办公区空调系统监控与优化的实际场景，完整展示DFS如何帮助企业实现：

- **实时监控**：多源数据融合与可视化
    
- **智能诊断**：异常检测与根因分析
    
- **预测优化**：基于数据的主动决策
    
- **运维闭环**：从发现问题到解决问题的完整流程
    

### 1.2 目标受众

- 设施管理负责人
    
- 楼宇运维团队
    
- 企业行政决策者
    
- 数字化转型负责人
    

### 1.3 演示时长

总时长：25分钟（含互动问答）

## 2. 演示场景设定

### 2.1 场景背景

某科技园区行政部面临两大挑战：

1. 频繁收到员工关于会议室温度不适的投诉
    
2. 夏季空调能耗持续攀升，超出预算
    

### 2.2 关键角色

- **运维工程师**：负责空调系统日常维护
    
- **设施经理**：负责整体能效管理
    
- **行政主管**：关注员工满意度
    

### 2.3 涉及区域

- 101会议室（东侧，早晨日照强烈）
    
- 102会议室（西侧，午后日照强烈）
    
- 大型报告厅（使用频率不规律）
    
- 空调机组A（负责东侧区域）
    
- 空调机组B（负责西侧区域）

## 4. 详细演示流程

### 4.1 开场引入（3分钟）

**主持人话术：**

"各位同事，想象一下这样的场景：周一早晨9点的101会议室，员工反映太冷；而下午3点的102会议室，却有人抱怨太热。行政部每天收到大量此类投诉，运维团队疲于奔命。今天，我们将展示如何通过DFS彻底改变这一现状。"

**操作：**

1. 展示投诉工单统计图表
    
2. 引出传统运维模式的痛点
    
3. 介绍DFS解决方案的价值主张
    

### 4.2 数字孪生场景展示（3分钟）

**操作流程：**

1. 在FactVerse Designer中打开办公区场景
    
2. 展示各区域的初始状态（全部显示为绿色，表示舒适）
    
3. 解释孪生体与物理设备的对应关系
    

**关键展示点：**

- 3D场景的真实感
    
- 设备与孪生体的——对应
    
- 实时数据驱动的可视化效果
    

### 4.3 多源数据接入演示（4分钟）

**操作流程：**

1. 切换到DFS平台【适配器管理】
    
2. 展示预配置的Node-RED数据流
    
3. 解释不同数据源的接入方式：
    
    - IoT传感器（MQTT）
        
    - 楼宇自控系统（BACnet）
        
    - 天气API（HTTP）
        
    - 能耗管理系统（数据库）
        
    

**数据流展示：**

```
传感器数据 → 数据清洗 → 格式转换 → 发布到DFS
天气数据 → 数据丰富 → 时序对齐 → 发布到DFS  
空调数据 → 状态计算 → 效率分析 → 发布到DFS
```

### 4.4 实时监控与异常发现（5分钟）

**看板布局：**

- **左上**：3D场景缩略图 + 关键指标卡
    
- **右上**：温度趋势对比图（今日vs昨日）
    
- **左下**：设备状态监控面板
    
- **右下**：预警信息列表
    

**演示脚本：**

"现在让我们关注空调系统监控中心。注意101会议室的颜色正在从绿色变为红色，同时系统弹出了预警信息。"

**异常触发时序：**

1. T+0：101会议室温度开始异常上升
    
2. T+2min：温度超过阈值，触发黄色预警
    
3. T+4min：温度持续上升，转为红色告警
    
4. T+5min：系统自动生成维修工单
    

### 4.5 根因分析深度探索（5分钟）

**分析步骤：**

1. **下钻到数据探索模块**
    
    - 选择预置的"空调效率分析"模板
        
    - 设定分析时间范围（异常发生前后2小时）
        
    
2. **多维度关联分析**
    
    ```
    # 分析维度
    维度1：101会议室温度 vs 空调机组A功率
    维度2：室内温度 vs 室外温度变化
    维度3：设备效率指标（COP）历史对比
    维度4：不同时段温度波动分析
    ```
    
3. **洞察发现**
    
    - 展示四线对比图（温度、功率、室外温度、效率）
        
    - 突出显示关键发现：
        
        - 空调机组效率下降30%
            
        - 101会议室午后温度波动异常
            
        - 能耗比去年同期增加15%
            
        
    

**诊断结论：**

"数据告诉我们两个问题：第一，空调机组A需要维护；第二，101会议室的西晒问题需要建筑层面的优化。"

### 4.6 预测性优化演示（3分钟）

**预测场景：**

1. **基于天气预报的负荷预测**
    
    - 接收明日高温预警（35℃）
        
    - 自动运行模拟任务
        
    - 生成优化运行策略
        
    
2. **优化建议展示：**
    
    ```
    明日运行建议：
    - 东侧会议室：上午10点开始预冷
    - 大型报告厅：设定温度调整为23℃
    - 空调机组A：建议安排维护窗口
    ```
    
3. **能效优化模拟**
    
    - 展示不同策略下的能耗对比
        
    - 计算预计节省金额
        
    

### 4.7 运维闭环展示（2分钟）

**工单流程：**

1. 预警自动生成维修工单
    
2. 工单分配给相应运维人员
    
3. 维修过程记录（照片、备注）
    
4. 完成后系统验证效果
    

**效果验证：**

- 展示维修后数据对比
    
- 温度恢复正常范围
    
- 能耗显著下降
