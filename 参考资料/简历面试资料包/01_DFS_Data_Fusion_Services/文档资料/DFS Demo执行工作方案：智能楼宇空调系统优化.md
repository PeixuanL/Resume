### 二、 阶段一：场景设计与资源准备

**目标：**​ 明确所有所需资源，并启动美术资产制作。

#### 任务1.1：明确模型需求（客成团队 -> 美术团队）

**交付物：**​ 详细的3D模型需求文档。

**需求内容：**

1. **办公区场景模型：**
    
    - **范围：**​ 包含至少3个独立空间：101会议室（标牌清晰）、102会议室、大型报告厅。一个开放的办公区，一个包含空调机组A和B的设备间或楼顶区域。
        
    - **细节：**​ 包含基本办公家具（桌椅、投影仪）、门窗、通风管道示意模型。
        
    - **格式：**​ FBX或GLTF，可导入FactVerse Designer。
        
    
2. **设备模型：**
    
    - **空调室内机：**​ 2-3种不同类型，用于不同房间。
        
    - **空调室外机组：**​ 2台，标注A和B。
        
    - **温湿度传感器：**​ 小型墙装设备模型。
        
    - **格式：**​ FBX或GLTF，模型原点设置合理。
        
    

#### 任务1.2：数据架构规划（客成团队）

**交付物：**​ 数据点表（Point Schedule）。

**规划内容：**==以下是样例==

|设备/数据点名称|类型|关键属性/指标|说明|
|---|---|---|---|
|`sensor_room_101`|虚拟IoT设备|`temperature`（温度）|101会议室温度传感器|
|`sensor_room_102`|虚拟IoT设备|`temperature`（温度）|102会议室温度传感器|
|`ahu_zone_east`|虚拟空调设备|`power`（功率）， `valve`（水阀开度）|东区空调机组|
|`weather_api`|虚拟数据源|`outdoor_temp`（室外温度）|模拟天气数据|

### 三、 阶段二：FactVerse Designer 场景搭建

**目标：**​ 构建用于Demo的数字孪生场景。

#### 任务2.1：导入模型与构建场景

1. **操作：**​ 在FactVerse Designer中创建新场景，如“DFS_Demo_Office”。
    
2. **操作：**​ 将美术团队交付的办公区模型导入场景，调整位置和比例。
    
3. **操作：**​ 将空调、传感器等设备模型放置在场景中相应位置。
    

#### 任务2.2：创建数字孪生体模板

1. **操作：**​ 创建“会议室”模板。
    
    - 添加动态属性：`温度`（类型：Float）。
        
    - 添加逻辑：当`温度`属性变化时，改变会议室内某个物体（如墙面LED屏幕）的颜色（蓝-绿-红渐变）。
        
    
2. **操作：**​ 创建“空调机组”模板。
    
    - 添加动态属性：`运行状态`（类型：String）、`当前功率`（类型：Float）。
        
    - 添加逻辑：当`运行状态`为“故障”时，模型显示红色告警动画。
        
    
3. **操作：**​ 从模板生成孪生体实例，如“101会议室孪生体”、“空调机组A孪生体”，并与场景中的模型进行绑定。
    

#### 任务2.3：发布场景

- **操作：**​ 完成场景搭建后，点击【发布】。记录生成的**场景ID**，后续在DFS中需要用到。
    

### 四、 阶段三：DFS 平台配置与数据接入

**目标：**​ 在DFS中完成数据流配置和设备绑定。

#### 任务3.1：导入数字孪生场景

1. **操作：**​ 登录DFS平台，进入【孪生体关联】>【场景配置】。
    
2. **操作：**​ 点击【导入】，从列表中选择在Designer中发布的“DFS_Demo_Office”场景。导入后，可在列表中看到场景及其包含的孪生体。
    

#### 任务3.2：准备演示数据

**交付物：**​ 可运行的MQTT数据模拟脚本（Python或Node-RED）。

**数据脚本要求：**

- **时序设计：**​ 脚本应能模拟**超30分钟**的数据，前10分钟正常，之后触发101会议室温度异常。
    
- **数据格式：**​ 严格按照DFS要求的JSON格式发布到MQTT主题。
    
    ```
    // 传感器数据示例（发布到 topic: dfs/demo/sensor）
    {
      "serial": "sensor_room_101",
      "ts": 1738407060000,
      "datas": {
        "temperature": 26.5
      }
    }
    // 空调数据示例（发布到 topic: dfs/demo/ahu）
    {
      "serial": "ahu_zone_east", 
      "ts": 1738407060000,
      "datas": {
        "power_consumption": 15.2,
        "water_valve": 95
      }
    }
    ```
    

#### 任务3.3：配置DFS适配器（Node-RED）

1. **操作：**​ 在DFS平台，【DFS适配器】>【适配器实例】中，【新建】一个实例，配置好IP和端口（如1880）。
    
2. **操作：**​ 访问该实例的Node-RED编辑器（[http://[ip]:1880](http://[ip]:1880/)）。
    
3. **操作：**​ 配置Node-RED流：
    
    - **MQTT输入节点：**​ 订阅上述演示数据脚本发布的Topic（`dfs/demo/#`）。
        
    - **功能节点（可选）：**​ 如需简单的数据清洗或转换。
        
    - **MQTT输出节点：**​ 将处理后的数据发布到DFS内部Topic（如`/DFS/telemetry/demo`），确保数据能被DFS后端接收。
        
    

#### 任务3.4：绑定设备与孪生体

1. **操作：**​ 进入【孪生体关联】>【设备绑定】。此时应能看到通过Node-RED流入DFS的设备（如`sensor_room_101`）。
    
2. **操作：**​ 为每个设备【配置孪生体】。
    
    - 选择场景“DFS_Demo_Office”。
        
    - 将设备`sensor_room_101`与孪生体“101会议室孪生体”绑定。
        
    - **属性绑定：**​ 将设备的`temperature`属性与孪生体的`温度`属性绑定。
        
    

### 五、 阶段四：数据分析与看板配置

**目标：**​ 配置数据探索任务和监控看板，以支撑Demo故事线。

#### 任务4.1：创建数据探索任务

1. **操作：**​ 在【数据探索】页面，【新建】任务。
    
2. **操作：**​ 选择数据源（即接入的会议室和空调数据）。
    
3. **操作：**​ 在画布上拖拽节点，构建分析流：
    
    - `原始数据`-> `特征提取（最大值、均值）`-> `结果输出`。
        
    - 配置一个节点，计算空调机组的COP（假设制冷量恒定，COP = 制冷量 / 功率）。
        
    
4. **操作：**​ 【保存】探索任务。此任务用于Demo中分析效率下降问题。
    

#### 任务4.2：配置数据看板

1. **操作：**​ 在【数据集管理】中，找到对应数据集，点击【数据看板】。
    
2. **操作：**​ 创建名为“空调系统监控中心”的看板。
    
3. **操作：**​ 添加并配置以下组件：
    
    - **统计指标：**​ 显示“平均温度”、“总实时功耗”。
        
    - **折线图：**​ 展示“101会议室温度（今日vs昨日）”。
        
    - **折线图：**​ 展示“空调机组A功率”和“COP值”。
        
    - **预警列表：**​ 配置规则，如“当101会议室温度 > 26℃时，触发预警”。
        
    

### 六、 阶段五：集成测试与Demo演练

**目标：**​ 确保整个流程万无一失。

#### 任务5.1：端到端集成测试

1. **测试用例1：数据驱动孪生体**
    
    - **步骤：**​ 运行数据模拟脚本 -> 打开Designer中的场景并播放。
        
    - **预期结果：**​ 101会议室的颜色随脚本中温度数据的变化而正常变化。
        
    
2. **测试用例2：看板预警**
    
    - **步骤：**​ 当温度超过26℃时，观察DFS看板。
        
    - **预期结果：**​ 看板预警列表正确出现报警信息。
        
    
3. **测试用例3：数据探索**
    
    - **步骤：**​ 执行预置的数据探索任务。
        
    - **预期结果：**​ 能正确计算出COP值，并展示效率下降趋势。
