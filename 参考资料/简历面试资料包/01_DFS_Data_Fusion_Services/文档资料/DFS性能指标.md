### 一、 实时性指标
1.  **端到端反向控制时延 (Control Loop Latency)**
    *   **定义：** 业务应用下发指令 -> 中台 -> 网关/边缘端 -> PLC/设备执行完成并返回ACK的时间。
    *   **建议值：**
        *   **L1级（实时控制，如紧急停机）：** < 50ms（通常需在边缘端完成，中台需监控边缘处理速度）。
        *   **L2级（参数下发，如修改机床参数）：** < 500ms。
        *   **L3级（看板监控）：** < 2秒。
2.  **告警触发实时性 (Alarm Trigger Latency)**
    *   **定义：** 设备异常数据产生 -> 规则引擎计算 -> 生成告警单的时间。
    *   **建议值：** P99 < 1秒。

### 二、 高频吞吐与采集指标
工业制造中，为了做故障预测（PHM）或质量检测，需要采集高频震动或电流波形数据。
1.  **高频采样支持能力 (High-Frequency Sampling Support)**
    *   **定义：** 单个连接通道能支持的最大采样频率。
    *   **建议值：** 支持 **1kHz - 10kHz**（即每秒上报1万个点）。注意，这通常通过打包压缩（Batching）上传，通过解析耗时来衡量。
2.  **协议解析速率 (Protocol Parsing Efficiency)**
    *   **定义：** 将原始二进制/Modbus/OPC UA数据流转换成标准JSON/数据模型的速率。
    *   **建议值：** 单节点解析能力 > 5万 TPS（Events/sec）。

### 三、 数据完整性与准确性
工业生产要做良率分析和OEE（设备综合效率）计算，数据的一致性比并发量更重要。
1.  **数据完整性 (Data Completeness)**
    *   **定义：** （实际落库的数据点数 / 理论应采集的数据点数）* 100%。
    *   **建议值：** **99.99% 以上**。
2.  **时钟同步精度 (Time Synchronization Accuracy)**
    *   **定义：** 设备上报时间戳与真实标准时间的误差。
    *   **建议值：** < 100ms（跨车间设备协同分析）；< 10ms（同产线设备关联分析）。
3.  **乱序处理窗口 (Out-of-Order Handling Window)**
    *   **定义：** 允许数据晚到多久还能被正确排序和计算。
    *   **建议值：** 支持至少 **5-10分钟** 的乱序重排窗口。

### 四、 边云协同性能
1.  **边云断网续传能力 (Edge-Cloud Resume Capability)**
    *   **定义：** 工厂断网后，边缘节点能缓存多久的数据？恢复网络后，多久能把积压数据传完？
    *   **建议值：** 边缘端至少缓存 **24小时** 数据；网络恢复后，积压数据同步速率应达到正常速率的 **3倍以上**。
2.  **配置下发同步时间 (Config Sync Latency)**
    *   **定义：** 在中台修改采集策略（如：把温度采集从1分钟改为10秒），同步到边缘网关生效的时间。
    *   **建议值：** < 30秒。

### 五、 存储成本与查询性能（核心：性价比）
工业数据量极大，且分为热数据（实时监控）、温数据（近期分析）、冷数据（合规存档）。
1.  **时序数据库压缩比 (Compression Ratio)**
    *   **定义：** 工业时序数据的磁盘占用压缩比。
    *   **建议值：** 针对浮点数（Float）和重复值，压缩比应达到 **15:1** 甚至 **20:1**。
2.  **聚合查询响应 (Aggregation Query Latency)**
    *   **最新值查询时延**：查询指定设备（或一批设备）**当前最新状态（如：现在的温度、现在的转速）所需的时间。
	    *  **指标：** 
		    * **P99 < 20ms** （热数据，通常在内存或缓存中）。
		    * **并发支持：** 支持 > 1000 QPS（即支持1000个面板组件同时刷新）。
	* **历史趋势降采样查询**：数据库在服务端将100万个点**降采样**（如：每小时取最大值/平均值）成500个点，并返回给前端的耗时。
		* **指标**：
			* **查询时间跨度1天（秒级数据）：** < 200ms。
			* **查询时间跨度1个月（分钟级数据）：** < 1秒。 
			* **查询时间跨度1年（小时级数据）：** < 3秒。
	* 原始数据导出性能：
		*  **指标：** 
			* **读取速率：** > 20 MB/s 或 > 500万点/秒
	* **多维聚合分析查询**：跨设备、跨时间段、带过滤条件的聚合计算耗时。
		* **指标：** * **百万级数据点聚合：** < 3秒。 * **千万级数据点聚合：** < 10秒（视硬件配置而定，通常涉及MPP处理）。
	* **冷热数据查询差异性能**：针对不同存储介质（SSD vs HDD/对象存储）的查询SLA区分。
		* **指标：**
			* **热数据（近7天）：** 毫秒级返回。
			* **冷数据（超过3个月）：** 允许 5-10秒 返回，甚至异步返回。
