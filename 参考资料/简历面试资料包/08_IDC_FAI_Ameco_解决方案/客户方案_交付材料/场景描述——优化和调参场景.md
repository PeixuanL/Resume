# 一、 MVP 目标
1.  **降门槛**：用户无需深研手册，即可通过对话完成基础的 3D 布局。
2.  **增效率**：将繁琐的行为树（BT）节点拖拽和参数配置，简化为自然语言描述。
3.  **验证架构**：打通 OpenClaw 的 Workspace/Memory 机制与 Designer 的 API 闭环。

# 二、 核心架构定义

## 2.1 工作空间 (Workspace) - AI 的“视网膜”
AI 在对话时实时读取的场景快照（JSON 格式），包含：
*   **Scene Snapshot**：
    *   所有物体的 `EntityID`、`TemplateName`、`Position(x,y,z)`、`Rotation`。
    *   物体间的层级关系（Parent-Child）。
    *   物体的出入口端口（Ports）定义。
*   **BT Snapshot**：
    *   当前画布上所有节点的 ID、类型、连接状态及参数配置。
*   **Template Dictionary**：
    *   当前租户下已加载的所有“孪生体模板”及其包含的属性、动画列表。

## 2.2 记忆 (Memory) - AI 的“大脑”
*   **短期记忆 (Session)**：记录本次会话中用户提及的“它”、“这个”、“刚才那个物体”。
*   **长期记忆 (Knowledge)**：
    *   **手册记忆**：用户手册第 4、5、6 章（快速上手、功能说明、行为树）。
    *   **逻辑模版记忆**：公司标准的“生成-传输-存储”经典行为树范例。
    *   **操作偏好**：例如用户习惯将传送带高度设为 0.8 米。

### 为什么“优化与调参”是当前 AI 的最佳切入点？
1.  **精度与效率瓶颈**：在 3D 编辑器中，从 0 搭建时，用户通过鼠标拖拽（所见即所得）的直观性远超通过语言描述空间位置。AI 往往难以一次性理解用户心中复杂的空间布局需求。
2.  **“调参”是数字孪生的核心痛点**：数字孪生的精髓在于“模拟真实”。用户往往已经搭建好了产线，但由于属性极其繁多（如 50 段传送带的速度、20 台机械臂的间隙、BT 中上百个判定逻辑），**手动逐一调整这些参数以达到模拟平衡点极其痛苦**。
3.  **AI 的长处在于“上下文理解”与“批量处理”**：AI 擅长在既有数据的基础上进行逻辑推理。例如：“根据目前的拥堵情况，自动微调所有传送带的速度”。

因此，我将重新调整场景重点，将 **“现有场景的持续优化与精细化调参”** 作为最高优先级场景进行细化。
以下侧重于 AI 如何理解现有场景的“亚健康”状态或“待调优”参数，并通过自然语言交互引导用户完成高效微调。

### 产品经理对细化场景的总结：

1.  **“预览-执行”闭环**：在上述场景中，AI 助手的行为始终遵循“**识别意图 -> 计算差值 -> 视觉预览（虚影/高亮）-> 用户确认 -> 最终写入**”的流程，避免了 AI 误操作带来的场景崩溃风险。
2.  **减少输入成本**：对话中大量使用了“选中的物体”、“调大一点”、“同步前缀”等模糊性词汇，AI 结合 Workspace 的上下文（当前选了谁、当前值是多少）将其转化为精确的 API 参数，这正是调参场景下 AI 的核心价值。
3.  **多助手联动**：例如架构师调完了位置，质检官主动介入检查冲突。这种协作体现了 OpenClaw 架构中独立 Assistant 又共享 Workspace 的优势。



# 六、产品形态：
### 1. 界面形态：从“全屏参数框”到“双驾驶员座舱”

- ==**左侧/中心（3D 场景与行为树编辑区）**：保持现有的“所见即所得”视图，但在物体上方会出现动态的“AI 建议层”。例如，当你通过 AI 微调位置时，3D 空间中会直接出现蓝色的虚影（Ghosting），预示修改后的位置，而不是直接生硬地挪动。==
    
- ==**右侧（侧边 AI 对话面板）**：这是产品的“大脑”。它采用 OpenClaw 架构，允许用户像切换频道一样切换助手（架构师、数据专家等）。==
    
- ~~**智能属性栏（Magic Bar）**：属性栏不再只是静态数值。当 AI 检测到某个参数（如速度、延迟）偏离了模拟最佳值时，参数框会轻微闪烁或变色，并附带一个小气泡：“AI 建议调至 1.5”。~~
    

### 2. 交互逻辑：从“指令式操作”到“意图式协作”

- ==**多模态融合**：用户视角的操作是混合的。你可以左手按住 Shift 框选场景中的 20 台电机，右手直接在侧边栏输入：“把选中的这些家伙全部对齐，并把电量阈值微调到 15%”。==
    
- ==**参数滑块生成（Interactive UI）**：当你对 AI 说“微调传送带速度”时，AI 助手不会只回文字，它会在对话框里直接生成一个**临时调参卡片**，上面有滑块。你滑动滑块，3D 场景里的 Twin 属性实时改变，这比反复对话确认要快得多。==
    
- ==**“Undo/Redo”的口语化**：你可以直接对 AI 说：“撤销刚才那次批量调参，还是原来那个版本好”，AI 会自动回溯 Workspace 的历史记录。==
    

### 3. 核心功能形态：场景的“自愈”与“调优”

- **主动审计（Proactive Audit）**：当你修改了行为树的一个关键逻辑，导致下游某个设备永远无法启动时，**场景质检官**助手会在侧边栏主动弹出一个气泡：“嘿，你刚才改的电量判定逻辑可能导致循环死锁，需要我帮你修正吗？”
    
- **仿真闭环调参**：这是产品最核心的形态。你运行了一遍“模拟记录”，点击“一键优化”。AI 助手会列出三条调参建议：
    1. “传送带 2 速过快，导致碰撞频发，建议降至 1.2m/s”
    2. “暂存区 A 容量不足，建议位置向左偏移并扩容”
    3. “行为树 5 号节点判定太严格，导致 AGV 闲置”  
        你勾选这三项，点击“全部应用”，场景瞬间完成“自愈”。
        

### 4. 记忆形态：越用越懂你的“私教”

- **知识库实时对齐**：产品形态中集成了一个“透明手册”。你操作任何功能，AI 都会在后台自动索引对应的手册章节。当你报错时，它给出的不是错误代码，而是：“参考手册 6.2.3 章节，你的属性名写错了”。
    
- **企业习惯沉淀**：长期使用后，AI 记住了你们公司的标准。例如，你们公司的 AGV 路线点永远是绿色的，传送带标准高度永远是 0.8 米。下次当你放下一个新设备时，AI 会说：“已按贵司惯例自动设置了位姿和逻辑”。
