根据 OpenClaw 的架构设计要求，我将 **FactVerse Designer AI 助手** 的架构定义（Workspace/Memory）与具体的业务行为描述（Behavioral Descriptions）分为两个独立章节进行深度细化。

# 第一章：工作空间 (Workspace) 与记忆 (Memory) 定义

本章节定义了 AI 助手在执行任务时能够“看到”什么（工作空间）以及“记得”什么（记忆），这是实现精准辅助的基础。

## 1.1 孪生场景架构师 (Scenario Architect)
*   **Workspace (实时上下文)**：
    *   **3D 场景数据**：当前场景所有孪生体实例的层级树、空间位姿（$x, y, z, r$）、缩放比例。
    *   **行为树拓扑**：当前打开的行为树 JSON 结构、节点连接关系、当前选中的节点。
    *   **元数据字典**：当前编辑模板的属性定义（静态/动态）、预设结构（入口/出口）。
    *   **资源库索引**：可用的 3D 模型、小工具及预设模板清单。
*   **Memory (知识与偏好)**：
    *   **短期记忆**：记录用户最近选中的 3 5 个物体、最近修改过的 3 个 BT 逻辑分支。
    *   **长期记忆**：用户常用的布局间距（如：传送带对接间距）、习惯性的逻辑命名规范（如：以 `Flow_` 开头的节点名）。

## 1.2 数智融合专家 (Data & Analytics Assistant)
*   **Workspace (实时上下文)**：
    *   **属性映射表**：当前场景内所有孪生体的 `Identifier` 与其绑定的 DFS 数据源路径。
    *   **数据流清单**：外部接入的 IoT 信号列表、Excel 导入的列名。
    *   **分析数据集**：最近一次模拟回顾生成的碰撞日志、位置时间序列数据、性能指标（FPS/DrawCall）。
*   **Memory (知识与偏好)**：
    *   **长期记忆**：企业内部数据字典（如：`Temp_01` 对应 `第一工区温度`）、常见的异常阈值定义。

## 1.3 场景质检官 (Quality & Audit Assistant)
*   **Workspace (实时上下文)**：
    *   **物理引擎参数**：碰撞检测开启状态、安全范围盒（Safety Box）尺寸。
    *   **连通性图谱**：路径模式（Path Mode）中的关键点连通性矩阵。
    *   **逻辑闭环状态**：BT 节点的输入输出完整性校验结果。
*   **Memory (知识与偏好)**：
    *   **长期记忆**：工业安全标准（如：人机共存区域的最小间距）、场景优化最佳实践规则库。

## 1.4 数字化内训向导 (Training & Help Assistant)
*   **Workspace (实时上下文)**：
    *   **课件逻辑区**：当前课件的步骤列表、触发条件配置、评分规则表。
    *   **UI 聚焦状态**：用户当前鼠标悬停或正在操作的编辑器面板。
*   **Memory (知识与偏好)**：
    *   **长期记忆**：完整版《FactVerse Designer 用户手册》索引、过往常见考题模板逻辑。

# 第二章：用户场景与助手行为描述

本章节详细描述 AI 助手在侧边对话框中如何响应用户请求，并转化为系统执行行为。

## 2.1 孪生场景架构师：空间与逻辑全编排

| 用户场景       | 用户输入 (侧边对话框)                                        | AI 行为描述 (助手响应)                                   | 系统执行 (API 调用)                                        |
| :--------- | :-------------------------------------------------- | :----------------------------------------------- | :--------------------------------------------------- |
| **快速线体搭建** | “在 $(0,0,0)$ 位置生成一条由 4 段 2 米传送带组成的 L 型产线，末端接一个暂存区。” | 解析坐标与模板，计算 4 段传送带的位姿及 90 度转角。                    | 调用 `BatchCreateEntity`；自动设置各段 `Length=2`。            |
| **智能对齐连接** | “把 AGV 小车移动到传送带出口，并建立逻辑连接。”                         | 获取传送带出口坐标，自动调整 AGV 朝向，建立端口关联。                    | 调用 `SetPosition`；调用 `ConnectPorts(Exit_1, Entry_1)`。 |
| **逻辑骨架生成** | “帮我写一段逻辑：当光电开关检测到货物，传送带延时 2 秒后停止。”                  | 在 BT 编辑器创建：条件节点(检测) -> 等待节点(2s) -> 属性设置节点(速度=0)。 | 调用 `AddBTNodes`；自动关联 Twin 的 `Speed` 属性。              |
| **逻辑重构优化** | “检查这个行为树，把所有冗余的等待节点合并。”                             | 扫描 BT 逻辑流，发现连续的等待节点，建议合并为单一节点。                   | 调用 `UpdateBTNode`；优化节点拓扑展示。                          |

## 2.2 数智融合专家：自动绑定与瓶颈诊断

| 用户场景 | 用户输入 (侧边对话框) | AI 行为描述 (助手响应) | 系统执行 (API 调用) |
| :--- | :--- | :--- | :--- |
| **批量数据绑定** | “根据这个 Excel 里的信号名，自动绑定场景里 20 台电机的频率属性。” | 模糊匹配 Excel 表头与孪生体 `Identifier`，提示匹配成功率。 | 批量调用 `SetDataBinding` API。 |
| **仿真瓶颈分析** | “分析昨天的模拟记录，找出为什么 5 号位经常撞车。” | 调取模拟记录中的碰撞事件（手册 5.9），定位冲突的时间点。 | 自动跳转模拟回顾快照；高亮显示冲突物。 |
| **性能优化方案** | “场景太卡了，帮我优化一下渲染。” | 扫描当前渲染参数，建议关闭非核心物体的“背面可见”或开启相机后处理。 | 执行 `ToggleBackfaceVisibility`；调用设置 API。 |

### 产品经理补充说明 (PM Insights)：
1.  **关于 API 开放**：为了支持“场景架构师”的复杂行为，我们需要重点开放 **`BehaviorTree_Seralize/Deserialize`**（支持 AI 读写 JSON 逻辑）和 **`Transform_BatchUpdate`**（支持空间阵列布局）。
2.  **关于私有化部署的准备**：虽然 demo 阶段不考虑，但在设计 API 时应遵循 RESTful 或 gRPC 标准，确保后续 LLM 插件（如 Local-LLM 或 RAG 引擎）可以平滑接入。
3.  **记忆的一致性**：OpenClaw 的 Memory 模块应支持跨助手共享，例如“架构师”刚放下的物体，“质检官”应立即意识到它的存在并进行审计。
